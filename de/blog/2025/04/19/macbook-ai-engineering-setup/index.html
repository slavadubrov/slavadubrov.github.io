Einrichtung eines MacBook für AI Engineering: macOS-Tools, Python, Docker und Terminal Workflow

Dies ist meine Checkliste, um eine frisch installierte macOS-Maschine in einen AI Engineering-Arbeitsrechner umzuwandeln: Befehlszeilenwerkzeuge, Homebrew, Python, Docker, Einstellungen der Terminal-Umgebung sowie Konfiguration des Editors.

TL;DR: Dies ist die Konfiguration, die ich auf einem neuen MacBook verwende: Homebrew, Python zusammen mit uv, das mitgelieferte Zsh inklusive Oh My Zsh und Powerlevel10k, Docker, meine Editor sowie lokale AI-Tools. Die Befehle sowie die Dotfiles werden im Versionssystem gespeichert, damit die Einrichtung immer wieder reproduzierbar ist.

1. Installation der Xcode Command Line Tools

xcode-select --install

Dies öffnet ein Dialogfeld, das Sie Schritt für Schritt durch die Installation führt.

2. Homebrew installieren

Homebrew ist der Paketmanager, den ich für alles andere in dieser Liste verwende:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Der Installer erläutert, welche Änderungen vorgenommen werden, und bittet um Bestätigung. Auf Apple Silicon verwendet Homebrew /opt/homebrew standardmäßig. Befolgen Sie brew shellenv Der Befehl wird am Ende ausgegeben; Homebrew stellt den Befehl nicht automatisch und ohne Anzeige in jeder Shell zur Verfügung.

3. Installation der Kernwerkzeuge

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

In groben Gruppen nach ihrem Verwendungszweck:

Python-Umgebung

Bevor ich uv eingeführt habe, nutzte ich pyenv als Grundlage für die Verwaltung meiner Python-Umgebung. Mittlerweile übernimmt uv diese täglichen Aufgaben Workflow für mich – einschließlich der Installation von Python sowie der Erstellung von Projektumgebungen.

  • UV installiert Python-Versionen und verwaltet Projektumgebungen, Abhängigkeiten, Befehle sowie Lockdateien. pyenv Dies ist optional. Ich verwende es nur dann, wenn ich einen aus Quellen selbst kompilierten Interpreter oder benutzerdefinierte CPython-Kompilierungsoptionen benötige, die in den vorkompilierten Distributionen von uv nicht vorhanden sind.

Installieren Sie pyenv separat, wenn Sie diesen Quellcode-Build-Pfad benötigen:

brew install pyenv

Systembibliotheken

  • openssl bietet Unterstützung für SSL/TLS-Kryptografie. Ein-/Ausgabezeile fügt die Textbearbeitung über die Kommandozeile hinzu. sqlite3 es handelt sich um eine leichte eingebettete Datenbank.
  • xz und zlib Bereitstellung von Datenkompression.

Produktivitätswerkzeuge

  • htop es handelt sich um einen Überwachungsmechanismus für visuelle Systeme und Prozesse. tmux verwaltet mehrere Terminals Sessions. ncdu Analysiert den Datenträgerverbrauch interaktiv.
  • Gitmoji fügt Emojis zu Commit-Nachrichten hinzu. pandoc konvertiert Dokumente zwischen verschiedenen Formaten.

Weitere Informationen dazu finden Sie unter uv sich selbst, siehe meinen Leitfaden zur Verwaltung von Python unter macOS mit uv.

4. Ein Terminal auswählen

Der Standard-Terminal unter macOS ist vollkommen ausreichend. Ich habe ihn verwendet iTerm2 seit Jahren und kürzlich umgezogen nach Verzerrung, ein auf Rust basierender Terminal mit integrierten AI-Funktionen. Die Wahl ist rein subjektiv; keine der anschließenden Konfigurationsschritte hängt davon ab.

Optional: Konfiguration von iTerm2

Wenn Sie weiterhin auf iTerm2 setzen, sind das die beiden Einstellungen, die ich stets anpasse:

Natürliche Textbearbeitung aktivieren

  1. Einstellungen → Profile → Schlüssel → Schlüsselzuordnungen
  2. Klicken Sie auf das Dropdown-Menü „Voreinstellungen…“
  3. Wählen Sie „Natürliche Textbearbeitung“ aus

Ein Farbthema auswählen

  1. Durchsuchen Sie Themen bei iTerm2-Farbschemata
  2. Einstellungen → Profile → Farben → Farbvorlagen…
  3. Klicken Sie auf „Importieren“ und wählen Sie das heruntergeladene Theme aus.

5. Einrichtung von Zsh

macOS enthält bereits Zsh und verwendet dieses als Standard-Anmeldeshell. Ich nutze die mitgelieferte /bin/zsh; installieren Homebrews Zsh nur dann, wenn Sie eine bestimmte, neuere Version aus der Quellentwicklung benötigen.

Überprüfen Sie den installierten Zsh sowie Ihre aktuelle Anmelde-SHELL:

echo "$SHELL"
command -v zsh
zsh --version

Falls Zsh installiert ist, aber nicht als Ihr Anmelde-Shell ausgewählt wurde, wechseln Sie auf die mitgelieferte Kopie:

chsh -s /bin/zsh

Öffnen Sie ein neues Terminal nach dem Ändern der Anmelde-SHELL.

Oh mein Zsh Es fügt die von mir verwendeten Standardwerte sowie das Plugin-System darauf hinzu. Sein offizieller Installer ist:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Das Projekt empfiehlt, einen unbekannten Installer vor dem Ausführen zu überprüfen. Es wird außerdem dokumentiert, dass ein bereits vorhandener … .zshrc wird in … umbenannt .zshrc.pre-oh-my-zsh während der Installation.

6. Hinzufügen von Zsh-Plugins

Ihren Code anpassen ~/.zshrc um diese Plugins zu laden:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

entfernen pyenv aus der Plugin-Liste, falls Sie die optionale Installation von pyenv übersprungen haben.

Die Beschreibungen für all diese befinden sich in dem Oh My Zsh Plugins Wiki.

Die letzten beiden erfordern eine separate Installation:

  • zsh-Autovorschläge empfiehlt Befehle aus Ihrer Historie, während Sie tippen. zsh-Syntax-Highlighting Farben werden während des Eingabens farblich hervorgehoben, wodurch Tippfehler leichter erkannt werden können.

Befolgen Sie die Installationsanleitungen in jedem Repository.

7. Thematisierung mit Powerlevel10k

Powerlevel10k es ist das Zsh-Thema, das ich verwende. Es zeigt den Arbeitsverzeichnis, den Git-Status sowie die aktive Python-Umgebung im Prompt an und verfügt über einen interaktiven Assistenten zur Anpassung des Aussehens. Die Installationsanleitungen finden Sie auf GitHub-Seite.

Schriftarten in anderen Editoren

Falls Sie VSCode oder einen anderen Editor mit integriertem Terminal verwenden, legen Sie die Schriftart des Terminals so fest, dass die Powerlevel10k-Ikonen korrekt dargestellt werden:

  1. Öffnen Sie die Einstellungen Ihres Editors.
  2. Suchen Sie nach … terminal.integrated.fontFamily
  3. Stellen Sie es darauf ein MesloLGS NF (Eingebettet zusammen mit Powerlevel10k)

Die vollständige Schriftartenkonfiguration befindet sich im Leitfaden zur Schriftart Powerlevel10k.

8. Editor und AI-Assistenten

Am Ende habe ich immer eine IDE geöffnet, neben der ein oder zwei AI-Tools laufen.

IDEs

  • Zeiger — ein VSCode Fork mit integriertem Pair-Programming für AI VS Code — der mit dem umfangreichen Erweiterungskatalog

AI Assistenten

  • OpenAI Codex — OpenAI’s Programmierung Agent
  • Claude — Anthropics Assistent – meine erste Anlaufstelle bei schwierigeren Aufgaben

9. Der Rest

Einige Zusatzkomponenten, die ich auf jedem Rechner installiere:

  • GitHub Desktop — visueller Git-Client, wenn ich nicht direkt in die CLI wechseln möchte Docker Desktop — der Container Runtime, den ich verwende; Podman Desktop ist eine Alternative
  • Ollama oder LM Studio — führen Sie LLMs lokal auf dem Mac aus.

10. Das ist die Einrichtung

Genau das installiere ich auf jedem neuen MacBook, bevor ich mit der eigentlichen Arbeit beginne. Passen Sie es nach Ihren Vorlieben an.

Wichtige Erkenntnisse

  1. Es handelt sich um eine persönliche Konfiguration und nicht um eine minimale oder universelle macOS-Basisversion. Entfernen Sie alles, was Sie nicht benötigen.
  2. Die mitgelieferte /bin/zsh dies ist für diese Konfiguration ausreichend. Installieren Sie nur dann Zsh von Homebrew, wenn Sie eine spezifische, neuere Version aus dem Quellcode benötigen.
  3. Mit uv werden nun die Python-Installationen sowie die Projektumgebungen in meinen täglichen Workflow. pyenv bleibt weiterhin optional, falls auf selbst kompilierte oder angepasste CPython-Interpreter zurückgegriffen wird.
  4. Dokumentieren Sie die Installationsbefehle, die nicht sensiblen Dotfiles, die Editor-Erweiterungen sowie die Model-Orte, damit eine spätere Konfiguration reibungslos verläuft.
