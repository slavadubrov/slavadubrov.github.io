    Einrichtung eines MacBook für die KI-Ingenieurwesen: macOS-Tools, Python, Docker und Terminal-Arbeitsabläufe

    Dies ist meine Checkliste, um eine frisch installierte macOS-Maschine in einen Arbeitsplatz für das KI-Ingenieurwesen umzuwandeln: Befehlszeilen-Tools, Homebrew, Python, Docker, Standardeinstellungen des Terminals sowie Einstellungen des Editors.

    Zusammenfassung: Eine praktische MacBook-Einrichtung für AI-Ingenieure ist absichtlich schlicht gehalten: Homebrew, uv, Docker oder Colima, Git, Shell-Konfigurationen, Editor-Tools, lokale LLM/OCR-Utilities sowie eine wiederholbare Sicherung der Dotfiles. Optimieren Sie die Einrichtung vor allem nach Geschwindigkeit bei der Neukompilierung – nicht nach Neuheit.

    1. Installation der Xcode Command Line Tools

    xcode-select --install

    Dies öffnet ein Dialogfeld, das Sie Schritt für Schritt durch die Installation führt.

    Homebrew ist der Paketmanager, den ich für alles andere in dieser Liste verwende:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    Der Installer fragt nach Ihrem Passwort und fügt sich selbst zu Ihren PATH## 3. Installation der Kernwerkzeuge

    brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

    Ungefähr nach ihrer Verwendungszweckgruppe eingeteilt:

    Python-Umgebung

    • pyenv — mehrere Python-Versionen nebeneinander verwalten UV — schneller Python-Paketmanager und Umgebungsverwalter

    Systembibliotheken

    • openssl — Unterstützung für SSL/TLS-Kryptographie Standard-Eingabezeile — Textbearbeitung über die Kommandozeile sqlite3 — leichte eingebettete Datenbank xz — Datenkompression zlib — Komprimierungsbibliothek

    Produktivitätswerkzeuge

    • htop — Monitor des visuellen Systems und Anzeige-Tool für Prozesse tmux — Verwaltung mehrerer Terminal-Sessions ncdu — interaktiv die Festplattennutzung analysieren gitmoji — Emojis zu Commit-Nachrichten hinzufügen
    • pandoc — Konvertierung von Dokumenten zwischen verschiedenen Formaten

    Weitere Informationen dazu finden Sie unter uv selbst, siehe meins Leitfaden zur Verwaltung von Python unter macOS mit uv## 4. Eine Terminal-Anwendung auswählen

    Das Standard‑Terminal von macOS ist vollkommen ausreichend. Ich habe es verwendet. iTerm2 seit Jahren und kürzlich umgezogen nach Warpung, ein auf Rust basierender Terminal mit integrierten KI-Funktionen. Sie können Warp direkt von deren Webseite### Optional: Einrichtung von iTerm2

    Falls Sie weiterhin auf iTerm2 setzen, sind dies die beiden Einstellungen, die ich stets anpasse:

    Aktivierung der natürlichen Textbearbeitung

    1. Preferences → Profiles → Keys → Key Mappings
    2. Klicken Sie auf das Dropdown-Menü „Presets…“
    3. Wählen Sie „Natural Text Editing“ aus

    Auswahl eines Farbthemas

    1. Durchsuchen Sie die verfügbaren Themen unter iTerm2-Farbschemata
    2. Einstellungen → Profile → Farben → Farbvorlagen…
    3. Klicken Sie auf „Importieren“ und wählen Sie das heruntergeladene Theme aus

    5. Zsh mit Oh My Zsh einrichten

    macOS verwendet standardmäßig Zsh. Oh mein Zsh fügt sinnvolle Standardwerte sowie ein Plugin-System hinzu:

    brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

    Der Installer erstellt einen Backup der vorhandenen Zsh-Konfiguration, bevor die neue Konfiguration eingeführt wird.

    6. Zsh-Plugins hinzufügen

    Bearbeiten Sie Ihre ~/.zshrc um diese Plugins zu laden:

    plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

    Die Beschreibungen für all diese befinden sich in dem Dokument. Oh mein Zsh-Plugins-WikiDie letzten beiden erfordern eine separate Installation:

    • zsh-Autovorschläge — schlägt Befehle vor, während Sie tippen, basierend auf Ihrer Historie zsh-Syntax-Highlighting — Die Befehle werden beim Eingeben farblich hervorgehoben, sodass Tippfehler sofort sichtbar sind.

    Befolgen Sie die Installationsanweisungen in jedem Repository.

    Powerlevel10k ist das Zsh-Thema, das ich verwende. Es zeigt im Prompt den Arbeitsverzeichnis, den Git-Status sowie die aktive Python-Umgebung an und verfügt über einen interaktiven Assistenten zur Anpassung des Aussehens. Die Installationsanleitungen finden Sie auf dem GitHub-Seite### Schriftarten in anderen Editoren

    Falls Sie VSCode oder einen anderen Editor mit integriertem Terminal verwenden, legen Sie die Schriftart des Terminals fest, damit die Powerlevel10k-Ikonen korrekt angezeigt werden:

    1. Öffnen Sie die Einstellungen Ihres Editors
    2. Suchen Sie nach terminal.integrated.fontFamily
    3. Stellen Sie es ein. MesloLGS NF (Eingebettet zusammen mit Powerlevel10k)

    Die vollständige Schriftartenkonfiguration befindet sich im Leitfaden zur Powerlevel10k-Schriftart## 8. Editor und KI-Assistenten

    IDEs

    9. Der Rest

    Einige Zusatzprogramme, die ich auf jedem Rechner installiere:

    • GitHub Desktop — visueller Git-Client, wenn ich nicht direkt in die CLI wechseln möchte Docker — Container; falls Sie es lieber nicht verwenden möchten, Podman und andere Alternativen den größten Teil desselben Stoffes abdecken Ollama oder LM Studio — LLMs lokal auf dem Mac ausführen

    10. So sieht die Einrichtung aus

    Das ist es, was ich auf jedem neuen MacBook vor der eigentlichen Arbeit installiere. Passen Sie es nach Ihren Vorlieben an.

    Wichtige Erkenntnisse

    1. Stellen Sie sicher, dass die Einrichtung reproduzierbar ist: Dokumentieren Sie die Installationsbefehle, Dotfiles, SSH-Schlüssel, Editor-Erweiterungen sowie die Orte für Modelle und Caches.
    2. Trennen Sie das System-Paketmanagement vom Python-Projektmanagement. Homebrew und uv lösen unterschiedliche Probleme.
    3. Installieren Sie lokale KI-Tools erst, nachdem die grundlegende Entwicklerumgebung stabil ist.
    4. Von der Qualität einer guten Arbeitsstation lässt sich ablesen, wie schnell Sie sie nach einem sauberen macOS-Install wieder aufbauen können.
