    Lokale LLM’s op macOS: Ollama, LM Studio, llama.cpp, MLX en Apple Silicon

    Het sturen van elke prompt naar een derdepartij-API wordt steeds vermoeiender, vooral wanneer de helft van de prompts bestaat uit verzoeken als “herschrijf dit paragraaf” of “wat is het JSON-schema hiervoor?”. Lokale LLM’s hebben dit probleem voor mij op Apple Silicon sneller opgelost dan ik had verwacht. Een 7B-model in 4-bit kwantisatie werkt soepel op een MacBook met 16 GB geheugen, en de hele werkwijze vindt plaats rechtstreeks via het toetsenbord.

    De open vraag is dus welk programma je gebruikt om ze te laten draaien. Ollama, LM Studio, llama.cpp, MLX en nog enkele andere tools maken allemaal gebruik van soortgelijke inferentie-engineën en dezelfde GGUF-bestanden. Ze verschillen vooral in de mate van complexiteit tussen de gebruiker en het model: aan de ene kant hoef je alleen maar dubbelklikken en typen; aan de andere kant moet je eerst vanuit broncode compileren en daarna de handleiding raadplegen.

    Kort samengevat: gebruik Ollama voor een eenvoudige lokale API, LM Studio voor een GUI en een OpenAI-compatibele endpoint, llama.cpp wanneer je directe controle nodig hebt over de inferentie van GGUF-bestanden, en MLX wanneer je Python-workflows wilt die specifiek zijn ontworpen voor Apple Silicon.

    Belangrijke concepten voor lokale LLM’s op macOS

    1. Inferentie: het uitvoeren van het model om tekst te genereren.
    2. Quantisatie (GGUF): een techniek om de grootte van het model te verkleinen met minimale kwaliteitsverliezen. Je zult bestandsnamen tegenkomen zoals llama-3-8b-Q4_K_M.gguf. De Q4 ‘Part’ duidt op 4-bits kwantisatie, waarbij veel minder RAM wordt gebruikt dan bij volledige 16-bits gewichten.
    3. Apple Silicon (Metal): De M-serie chips van Apple delen de RAM tussen de CPU en de GPU (Apple noemt dit “Unified Memory”). Daarom kan een MacBook met 32GB of 64GB modellen laden die op een PC een dure dedicated GPU zouden vereisen.

    Voorwaarden

    • Hardware: een Mac met Apple Silicon (van M1 tot M4) is wat je nodig hebt. Intel Macs werken ook, maar ze zijn aanzienlijk trager.
    • RAM:
      • 8GB: voldoende voor kleine modellen (Mistral 7B, Llama 3 8B).
      • 16GB+: ideaal voor grotere modellen en meerdere taken tegelijk uitvoeren.
    • Schijfruimte: modellen nemen veel ruimte in beslag. Reken op ongeveer 10-20GB voor een bruikbare startset met bibliotheken.

    1. Ollama - De “gewoon werken”-optie

    Download: ollama.com

    Denk aan Ollama als de "Docker voor LLM's". Het omhult de llama.cpp De engine zit in een native macOS-pakket, laadt modellen op verzoek en stuurt de verwerking automatisch door naar Metal. Je installeert het, voert één opdracht uit en je kunt al chatten. Dit is de eenvoudigste manier als je gewoon een werkende LLM en een HTTP-endpoint wilt die je code kan gebruiken.

    Voorbeeldwerkflow

    # 1. Download and run Llama 3 (it auto-downloads if needed)
ollama run llama3

# 2. Use it in your code via the local API
curl http://localhost:11434/api/generate -d '{
  "model": "llama3",
  "prompt": "Explain quantum computing to a 5-year-old",
  "stream": false
}'

    Voordelen en nadelen

    ✅ Voordelen ❌ Nadelen
    Eenvoudigste opstelling (dragen en neerzetten) .dmg) De kernapplicatie is gesloten broncode.
    Schone CLI (ollama list, ollama pull) Minder gedetailleerde controle over de generatieparameters
    Grote bibliotheek met vooraf geconfigureerde modellen

    2. LM Studio – De visuele explorer

    Downloaden: lmstudio.ai

    LM Studio is de optie die prioriteit geeft aan een grafische gebruikersinterface. Het beschikt over een browser in App-Store-stijl om rechtstreeks op HuggingFace te zoeken, ondersteunt Apple’s MLX-formaat naast GGUF (dat op sommige Macs sneller kan werken), en biedt een lokale server die compatibel is met OpenAI. Bestaande clientcode die al verbinding maakt met OpenAI werkt dan meestal gewoon door.

    Voorbeeldwerkflow

    Je kunt het officiële Python SDK gebruiken, of een willekeurige OpenAI-compatibele client die is ingesteld op de lokale server:

    # Using the official LM Studio SDK
from lmstudio import LMStudio

client = LMStudio()
response = client.complete(
    model="llama-3-8b",
    prompt="Write a haiku about debugging."
)
print(response.content)

    Voordelen en nadelen

    ✅ Voordelen ❌ Nadelen
    Een geoptimaliseerde, gebruiksvriendelijke interface De GUI is gesloten broncode.
    Natuurlijke ondersteuning voor zowel GGUF- als MLX-modellen Grootere download (~750 MB)
    Geïntegreerde RAG (chatten met uw PDF’s)

    3. llama.cpp – Het hulpmiddel voor geavanceerde gebruikers

    Repo: github.com/ggml-org/llama.cpp

    Dit is de engine die bijna elk ander hulpprogramma gebruikt als basis. Als u maximale prestaties, de nieuwste functies zodra ze beschikbaar komen, of wilt integreren van een LLM in uw eigen C++-toepassing, dan is dit de bron. Het werkt op bare metal en is lichtgewicht, maar het nadeel is dat u alles zelf moet beheren: downloads, formaten en tientallen CLI-vlaggen.

    Voorbeeldwerkflow

    # 1. Install via Homebrew
brew install llama.cpp

# 2. Download a model manually (e.g., from HuggingFace)
huggingface-cli download TheBloke/Llama-3-8B-Instruct-GGUF --local-dir .

# 3. Run inference with full control
llama-cli -m llama-3-8b-instruct.Q4_K_M.gguf \
  -p "Write a python script to sort a list" \
  -n 512 \
  --temp 0.7 \
  --ctx-size 4096

    Voordelen en nadelen

    ✅ Voordelen ❌ Nadelen
    Volledige controle over elke parameter Steile leercurve (alleen CLI).
    MIT-licentie (open source) Handmatige modellbeheer
    Zeer lichtgewicht (<30MB)

    4. GPT4All – Privacy voorop en RAG

    Downloaden: gpt4all.io

    GPT4All is gebaseerd op twee kernconcepten: privacy en documenten. De belangrijkste functie, LocalDocs, maakt het mogelijk om de app te richten op een map met PDF’s, notities of code, zodat je rechtstreeks met de inhoud kunt chatten. Alles werkt offline, zonder enige gegevensverzameling. Het is de eenvoudigste manier om een functionerende RAG-oplossing op je eigen machine te krijgen, zonder enige code te hoeven schrijven.

    Voordelen en nadelen

    ✅ Voordelen ❌ Nadelen
    LocalDocs RAG werkt uit de doos al zeer goed. Alleen GUI (geen headless-modus)
    Volledig offline en privé Hogere ressourcengebruik dan Ollama
    Cross-platform (Mac, Windows, Linux)

    5. KoboldCPP – Voor verhalenvertellers

    Repo: github.com/LostRuins/koboldcpp

    een fork van llama.cpp Gedacht voor creatief schrijven en RPG’s in tabletop-stijl. Het werkt als een lokale webapp met hulpmiddelen voor het genereren van lange teksten: “World Info”, karaktergeheugen en technieken om de consistentie van het verhaal te bewaren, zodat het model duizenden tokens lang op koers blijft. De doelgroep bestaat uit schrijvers en mensen die tekstgebaseerde RPG’s spelen; als je vooral chatten of programmeren wilt, zal de standaardinterface te beperkt aanvoelen.

    Voorbeeldwerkflow

    # 1. Download the single binary
wget https://github.com/LostRuins/koboldcpp/releases/latest/download/koboldcpp-mac.zip

# 2. Run it (launches a web server)
./koboldcpp --model llama-3-8b.gguf --port 5001 --smartcontext

    Voordelen en nadelen

    ✅ Voordelen ❌ Nadelen
    Krachtige hulpmiddelen voor creatief schrijven Niche UI (niet ideaal voor programmeren/chatten)
    Eénbestandse uitvoerbare (geen installatie nodig) AGPL-licentie (beperkend voor commercieel gebruik)

    Eervolle vermelding: MLX-LM

    Als je een Python-ontwikkelaar bent op Apple Silicon, kijk dan naar MLX-LM van Apple. Het is een framework dat is afgestemd op de M-serie chips, en op de juiste hardware is het vaak de snelste manier om een model lokaal uit te voeren. Het nadeel is dat het minder gebruiksvriendelijk is dan Ollama: meer Python, minder beperkingen.

    Wat je daadwerkelijk moet kiezen

    • Begin met Ollama als je gewoon iets wilt laten draaien vandaag.
    • Gebruik LM Studio als je eerst visueel door de modellen wilt bladeren.
    • Kies voor llama.cpp wanneer u volledige controle over de inferentie nodig heeft.

    Belangrijkste conclusies

    1. Ollama is de snelste manier om een lokale modelserver voor ontwikkeling te krijgen.
    2. LM Studio is het sterkst wanneer je een desktopinterface wilt en gemakkelijk modellen wilt onderzoeken.
    3. llama.cpp biedt de meest directe controle over GGUF-modellen en laaggelegen runtime-vlaggen.
    4. MLX is de geïntegreerde optie voor Apple Silicon wanneer je comfortabel bent met het schrijven van Python en het gebruiken van MLX-modelformaten.
