Instellingen voor MacBook voor AI-engineering: macOS-hulpmiddelen, Python, Docker en workflow in de terminal

Dit is mijn controlelijst om een nieuwe macOS-installatie om te zetten in een werkstation voor AI-engineering: hulpprogramma’s voor de commandoregel, Homebrew, Python, Docker, standaardinstellingen van de terminal en instellingen van de teksteditor.

TL;DR: Een praktische MacBook-instelling voor AI-engineering is opzettelijk eenvoudig: Homebrew, uv, Docker of Colima, Git, shell-configuratie, editor-hulpprogramma’s, lokale LLM/OCR-hulpmiddelen, en een herhaalbare back-up van dotfiles. Optimaliseer voor snelheid bij opbouw, niet voor nieuwigheid.

1. Installeer de Xcode Command Line Tools

xcode-select --install

Dit opent een dialoogvenster dat u stap voor stap door de installatie begeleidt.

Homebrew het is de pakketbeheerder die ik gebruik voor al het andere op deze lijst:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

De installer vraagt om uw wachtwoord en voegt zichzelf toe aan uw PATH## 3. Installeer de kernhulpmiddelen

brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

Ongeveer gegroepeerd naar hun doel:

Python-omgeving

  • pyenv — meerdere Python-versies naast elkaar beheren UV — snelle Python-pakketbeheerder en omgevingsbeheerder

Systeembibliotheken

  • openssl — SSL/TLS-cryptografische ondersteuning readline sqlite3 — lichte ingebouwde database xz — gegevenscompressie zlib — compressielibrairie

Productiviteitstools

  • htop — Monitor voor het visuele systeem en viewer voor processen tmux — Meerdere terminalsessies beheren ncdu — interactief de schijfgebruik analyseren gitmoji — Emoticons toevoegen aan commitberichten
  • pandoc — Converteer documenten tussen formaten

Voor meer informatie over uv zelf, zie het mijne Snelle handleiding voor het beheren van Python op macOS met uv.

4. Kies een terminal

De standaard macOS Terminal is prima. Ik gebruikte iTerm2 al jarenlang en onlangs verhuisd naar Warping, een terminal gebaseerd op Rust met ingebouwde AI-functies. Je kunt Warp van hun website### Optioneel: Instellingen van iTerm2

Als je bij iTerm2 blijft, zijn dit de twee instellingen die ik altijd aanpas:

Natuurlijke tekstbewerking inschakelen

  1. Preferences → Profiles → Keys → Key Mappings
  2. Klik op het dropdown-menu Presets…
  3. Kies “Natural Text Editing”

Een kleurenthema kiezen

  1. Blader door de thema’s op iTerm2-kleurschema’s
  2. Voorkeuren → Profielen → Kleuren → Kleurprestaties…
  3. Klik op Import en selecteer het gedownloade thema

5. Zsh instellen met Oh My Zsh

macOS gebruikt standaard Zsh. Oh mijn Zsh voegt gezonde standaardwaarden en een plugin-systeem toe:

brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

De installer maakt een back-up van uw huidige Zsh-configuratie voordat de nieuwe wordt geïnstalleerd.

6. Zsh-plugins toevoegen

Editeer uw ~/.zshrc om deze plugins te laden:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

De beschrijvingen voor al deze elementen staan in de Oh My Zsh plugins wiki.

De laatste twee vereisen een aparte installatie:

  • zsh-autosuggesties — suggereert commando’s terwijl je typt, gebaseerd op je historie zsh-syntaxis-highlighting — Kleurt de commando’s terwijl je typt, zodat typfouten duidelijk zichtbaar zijn.

Volg de installatie-instructies in elk repository.

7. Stijl het thema met Powerlevel10k

Powerlevel10k Dit is het Zsh-thema dat ik gebruik. Het toont de werkdirectory, de git-status en de actieve Python-omgeving in de prompt, en er is een interactieve wizard om het uiterlijk aan te passen. De installatie-instructies staan op de GitHub-pagina.

Schriftsoorten in andere editors

Als je VSCode of een andere editor met een geïntegreerde terminal gebruikt, stel dan de schriftsoort van de terminal in zodat de Powerlevel10k-icoontjes correct weergegeven worden:

  1. Open de instellingen van je editor
  2. Zoek naar terminal.integrated.fontFamily
  3. Stel het in op MesloLGS NF (gedownload samen met Powerlevel10k)

De volledige instelling van de schriftgroottes bevindt zich in de Powerlevel10k-lettertypegids.

8. Editor en AI-assistenten

Ik heb altijd minstens één IDE openstaan, met daarnaast één of twee AI-hulpprogramma’s die actief zijn.

IDE’s

  • Cursor — een VSCode-fork met ingebouwd AI-pairprogrammeren VSCode — degene met het uitgebreide catalogus van extensies

AI-assistenten

  • OpenAI Codex — Codegeneratie en completie via OpenAI Claude — De assistent van Anthropic; waar ik op terugval bij moeilijkere taken.

Tegenwoordig is dat Cursor met Codex en Claude Code die parallel draaien.

9. Verder nog

Een paar extra’s die ik op elk apparaat installeer:

  • GitHub Desktop — een visuele Git-client wanneer ik niet via de CLI wil werken Docker — containers; als u ze liever niet wilt gebruiken, Podman en andere alternatieven hetzelfde gebied grotendeels bedekken Ollama of LM Studio — LLM’s lokaal draaien op de Mac

10. Dat is de configuratie

Dit is wat ik installeer op elk nieuw MacBook voordat ik aan echt werk begin. Pas het naar eigen smaak aan.

Belangrijkste punten

  1. Zorg ervoor dat de configuratie reproduceerbaar is: documenteer de installatiecommando’s, dotfiles, SSH-sleutels, editor-extensies en de locaties van modellen en caches.
  2. Scheid het systeembeheer van pakketten van het beheer van Python-projecten. Homebrew en uv lossen verschillende problemen op.
  3. Installeer lokale AI-hulpmiddelen pas nadat de basisontwikkelaarsomgeving stabiel is.
  4. Een goede werkstationconfiguratie wordt gemeten aan de hand van hoe snel deze kan worden hersteld na een schone macOS-installatie.
