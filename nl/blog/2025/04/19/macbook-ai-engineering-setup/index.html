Instellingen voor een MacBook voor AI Engineering: macOS-hulpprogramma’s, Python, Docker en Terminal Workflow

Dit is mijn controlelijst om een nieuwe macOS-installatie om te zetten in een AI engineering werkstation: hulpprogramma’s voor de commandoregel, Homebrew, Python, Docker, standaardinstellingen van de terminal en instellingen van de teksteditor.

TL;DR: Dit is de configuratie die ik gebruik op een nieuwe MacBook: Homebrew, Python met uv, de geïnstalleerde Zsh met Oh My Zsh en Powerlevel10k, Docker, mijn teksteditoren, en lokale AI hulpprogramma’s. Zorg ervoor dat de commando’s en dotfiles onder versiebeheer vallen, zodat de configuratie steeds op dezelfde manier kan worden hersteld.

1. Installeer de Xcode Command Line Tools

xcode-select --install

Dit opent een dialoogvenster dat u stap voor stap door het installatieproces begeleidt.

2. Installeer Homebrew

Homebrew is de pakketbeheerder die ik gebruik voor al het overige op deze lijst:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

De installer legt uit wat er zal veranderen en vraagt om bevestiging. Op Apple Silicon maakt Homebrew gebruik van /opt/homebrew door defect. Houd u hieraan. brew shellenv Het commando wordt aan het einde weergegeven; Homebrew zorgt er niet automatisch voor dat dit commando in elke shell beschikbaar is.

3. Installeer de kernhulpmiddelen

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

Grotendeels gegroepeerd naar hun doel:

Python-omgeving

Voordat ik uv ging gebruiken, gebruikte ik pyenv als basis voor het beheren van mijn Python-omgeving. uv vervangt nu die dagelijkse workflow taak voor mij, inclusief de installatie van Python en het beheren van projectomgevingen.

  • UV Installeert Python-versies en beheert projectomgevingen, afhankelijkheden, commando’s en lockbestanden. pyenv Dit is optioneel. Ik gebruik het alleen wanneer ik een interpreter die vanaf nul is gebouwd of aangepaste CPython-configuratieopties nodig heb, die niet beschikbaar zijn in de vooraf gebouwde distributies van uv.

Installeer pyenv apart wanneer je die source-build-pad nodig hebt:

brew install pyenv

Systeembibliotheken

  • openssl Biedt ondersteuning voor SSL/TLS-cryptografie. readline Voegt tekstbewerking via de commandoregel toe. sqlite3 Het is een lichtgewicht ingebouwde database.
  • xz en zlib Zorg voor gegevenscompressie.

Productiviteitstools

  • htop het is een visueel systeem- en procesmonitor. tmux beheert meerdere terminals sessions. ncdu Analyseert het schijfgebruik op interactieve manier.
  • gitmoji voegt emoji’s toe aan commitberichten. pandoc Converteert documenten tussen verschillende formaten.

Voor meer informatie over uv zelf, zie mijn Snelle handleiding voor het beheren van Python op macOS met uv.

4. Kies een terminal

De standaard macOS Terminal is prima. Ik heb hem gebruikt iTerm2 al jarenlang, en onlangs verhuisd naar Warping, Een terminal gebaseerd op Rust met ingebouwde AI-functies. De keuze is persoonlijk; geen enkele latere configuratie is daarvan afhankelijk.

Optioneel: Instellingen van iTerm2

Als je bij iTerm2 blijft, zijn dit de twee instellingen die ik altijd aanpas:

Natuurlijke tekstbewerking inschakelen

  1. Voorkeuren → Profielen → Sleutels → Sleutelkoppelingen
  2. Klik op de dropdown ‘Vooraf ingestelde waarden…’
  3. Kies “Natuurlijke tekstbewerking”

Kies een kleurenthema

  1. Blader door thema’s iTerm2-kleurschema’s
  2. Voorkeuren → Profielen → Kleuren → Kleurprestaties…
  3. Klik op Importeren en selecteer het gedownloade thema.

5. Zsh instellen

macOS bevat al Zsh en gebruikt deze als de standaard inlogshell. Ik gebruik de meegeleverde /bin/zsh; installeren Homebrew’s Zsh Alleen wanneer u een specifieke, nieuwere upstream-versie nodig heeft.

Controleer de geïnstalleerde Zsh en je huidige inlogshell:

echo "$SHELL"
command -v zsh
zsh --version

Als Zsh geïnstalleerd is maar niet is geselecteerd als uw inlogshell, schakel dan over naar de meegeleverde versie:

chsh -s /bin/zsh

Opent een nieuwe terminal nadat de inlogshell is gewijzigd.

Oh mijn Zsh Het voegt de standaardinstellingen en het plugin-systeem toe die ik erbovenop gebruik. De officiële installer is:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Het project raadt aan om een onbekende installer te controleren voordat deze wordt uitgevoerd. Er wordt ook gemeld dat er al een bestaat. .zshrc wordt hernoemd tot .zshrc.pre-oh-my-zsh tijdens de installatie.

6. Zsh-plugins toevoegen

Uw … bewerken ~/.zshrc om deze plugins te laden:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Verwijderen pyenv uit de lijst met plugins, indien u de optionele installatie van pyenv hebt weggelaten.

De beschrijvingen voor al deze elementen staan in de Oh My Zsh plugins wiki.

De laatste twee vereisen een aparte installatie:

  • zsh-autosuggestions stelt op het moment dat je typt commando’s voor uit je geschiedenis. zsh-syntaxis-highlighting Kleuren worden weergegeven terwijl je typt, waardoor typfouten gemakkelijker op te merken zijn.

Volg de installatie-instructies in elk repository.

7. Stijl toevoegen met Powerlevel10k

Powerlevel10k dit is het Zsh-thema dat ik gebruik. Het toont de werkmap, de git-status en de actieve Python-omgeving in de prompt, en er is een interactieve wizard beschikbaar om het uiterlijk aan te passen. De installatie-instructies staan op de GitHub-pagina.

Schriftsoorten in andere editors

Als je VSCode of een andere editor met een geïntegreerde terminal gebruikt, stel dan de lettertype van de terminal in zodat de Powerlevel10k-icoontjes correct weergegeven worden:

  1. Ga naar de instellingen van je editor
  2. Zoek naar terminal.integrated.fontFamily
  3. Stel het in op MesloLGS NF (gedownload samen met Powerlevel10k)

De volledige fontinstellingen staan in de Powerlevel10k-lettertypegids.

Belangrijkste conclusies

  1. Dit is een persoonlijke instelling, geen minimale of universele macOS-basisinstelling. Verwijder alles wat u niet gebruikt.
  2. De meegeleverde /bin/zsh Dit is voldoende voor deze configuratie. Installeer alleen de Zsh van Homebrew wanneer je een specifieke, nieuwere versie van de broncode nodig hebt.
  3. uv beheert nu de installatie van Python en de projectomgevingen in mijn dagelijkse workflow. pyenv blijft optioneel voor handgemaakte of aangepaste CPython-interpreters.
  4. Houd de installatiecommando’s, dotfiles zonder gevoelige informatie, editorextensies en de model-locaties gedocumenteerd, zodat de volgende configuratie eenvoudig kan worden uitgevoerd.
