Configuration du MacBook pour l’ingénierie AI : outils macOS, Python, Docker et flux de travail via le Terminal

Voici ma liste de contrôle pour transformer une installation fraîche de macOS en poste de travail d’ingénierie AI : outils en ligne de commande, Homebrew, Python, Docker, paramètres par défaut du terminal et configuration de l’éditeur.

TL;DR : Voici la configuration que j’utilise sur un nouvel MacBook : Homebrew, Python accompagné de uv, le Zsh fourni avec Oh My Zsh et Powerlevel10k, Docker, mes éditeurs, ainsi que des outils locaux AI. Il est essentiel de placer les commandes et les fichiers de configuration sous contrôle de version afin de garantir que cette configuration puisse être reproduite à tout moment.

1. Installer les outils en ligne de commande Xcode

xcode-select --install

Cela ouvre une boîte de dialogue qui vous guide pas à pas lors de l’installation.

2. Installer Homebrew

Homebrew c’est le gestionnaire de paquets que j’utilise pour tout le reste de cette liste :

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

L’installateur explique quels éléments seront modifiés et demande une confirmation. Sur Apple Silicon, Homebrew utilise /opt/homebrew par défaut. Suivez brew shellenv Commande affichée à la fin ; Homebrew ne rend pas cette commande disponible automatiquement dans tous les shells sans aucun message.

3. Installer les outils de base

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

Groupés de manière approximative selon leur fonction :

Environnement Python

Avant d’adopter uv, j’utilisais pyenv comme base pour la gestion de mon environnement Python. Aujourd’hui, uv couvre entièrement ce flux de travail quotidien, y compris l’installation de Python et la création des environnements de projet.

  • UV installe les versions de Python et gère les environnements de projet, les dépendances, les commandes ainsi que les fichiers de verrouillage. pyenv C’est optionnel. J’utilise cette méthode uniquement lorsque j’ai besoin d’un interpréteur compilé à partir de la source ou de paramètres de compilation personnalisés pour CPython que les distributions précompilées d’uv ne proposent pas.

Installez pyenv séparément lorsque vous avez besoin de ce chemin de compilation depuis la source :

brew install pyenv

Bibliothèques système

  • openssl Fournit un soutien en matière de cryptographie SSL/TLS. lecture de ligne ajoute une fonction d’édition de texte en ligne de commande. sqlite3 Il s’agit d’une base de données embarquée légère.
  • xz et zlib Fournir une compression des données.

Outils de productivité

  • htop Il s’agit d’un moniteur de système visuel et de processus. tmux gère plusieurs sessions de terminal. ncdu Analyse de l’utilisation du disque de manière interactive.
  • gitmoji ajoute des emojis aux messages de commit. pandoc convertit les documents entre différents formats.

Pour en savoir plus sur uv en soi, voir mon Guide rapide pour la gestion de Python sous macOS à l’aide de uv.

4. Sélectionner un terminal

Le Terminal par défaut de macOS est tout à fait suffisant. J’ai utilisé iTerm2 pendant des années, et a récemment été transféré vers Warpage, Un terminal basé sur Rust doté de fonctionnalités intégrées AI. Le choix reste subjectif, car aucune des étapes de configuration ultérieures ne dépend de lui.

Optionnel : configuration d’iTerm2

Si vous continuez à utiliser iTerm2, voici les deux paramètres que je modifie systématiquement :

Activer l’édition de texte naturelle

  1. Préférences → Profils → Clés → Mappages de clés
  2. Cliquez sur le menu déroulant Préfixes…
  3. Sélectionnez « Édition de texte naturel »

Sélectionner un thème de couleur

  1. Parcourir les thèmes à iTerm2-Color-Schemes
  2. Préférences → Profils → Couleurs → Préférences de couleur…
  3. Cliquez sur « Importer » et sélectionnez le thème que vous avez téléchargé

5. Configuration de Zsh

macOS intègre déjà Zsh et l’utilise comme shell de connexion par défaut. J’utilise celui fourni avec le système. /bin/zsh; installer Zsh d’Homebrew uniquement lorsque vous avez besoin d’une version plus récente spécifique provenant du noyau source.

Vérifiez la version de Zsh installée ainsi que votre shell de connexion actuel :

echo "$SHELL"
command -v zsh
zsh --version

Si Zsh est installé mais n’a pas été sélectionné comme shell de connexion, passez à la copie fournie avec le logiciel :

chsh -s /bin/zsh

Ouvrez un nouveau terminal après avoir modifié le shell de connexion.

Oh Mon Zsh il ajoute par-dessus les valeurs par défaut ainsi que le système de plugins que j’utilise. Son installateur officiel est :

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Le projet recommande d’examiner un installateur inconnu avant de le lancer. Il précise également qu’un existant .zshrc est renommé en .zshrc.pre-oh-my-zsh pendant l’installation.

6. Ajouter des plugins Zsh

Modifier votre ~/.zshrc pour charger ces plugins :

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Supprimer pyenv dans la liste des plugins, si vous avez omis l’installation optionnelle de pyenv.

Les descriptions de tous ceux-ci se trouvent dans le Oh My Zsh plugins wiki.

Les deux derniers nécessitent une installation distincte :

  • zsh-autosuggestions Suggère des commandes issues de votre historique au fur et à mesure que vous tapez. zsh-syntax-highlighting Les commandes s’affichent en temps réel au fur et à mesure que vous tapez, ce qui facilite la détection des fautes d’orthographe.

Suivez les instructions d’installation indiquées dans chaque répertoire.

7. Thématiser avec Powerlevel10k

Powerlevel10k c’est le thème Zsh que j’utilise. Il affiche le répertoire de travail, l’état Git ainsi que l’environnement Python actif dans le prompt, et il dispose d’un assistant interactif permettant d’ajuster l’apparence. Les instructions d’installation se trouvent sur le page GitHub.

Polices dans d’autres éditeurs

Si vous utilisez VSCode ou un autre éditeur doté d’une terminal intégrée, configurez la police de caractères de la terminal afin que les icônes Powerlevel10k s’affichent correctement :

  1. Ouvrez les paramètres de votre éditeur
  2. Faites une recherche pour terminal.integrated.fontFamily
  3. Définissez-le sur MesloLGS NF (installé en même temps que Powerlevel10k)

Le paramétrage complet de la police se trouve dans le Guide de police Powerlevel10k.

8. Éditeur et assistants AI

Je finis systématiquement par avoir un IDE ouvert, avec un ou deux outils AI en cours d’exécution à côté de lui.

IDEs

  • Curseur — un VSCode fork intégrant la programmation en paire AI VSCode — celui qui dispose d’un vaste catalogue d’extensions

AI assistants

  • OpenAI Codex — L’agent de codage d’OpenAI Claude — L’assistant d’Anthropic ; l’outil auquel je recours pour les tâches les plus complexes

De nos jours, il s’agit de Cursor avec Codex et Claude Code fonctionnant en parallèle.

9. Le reste

Quelques éléments supplémentaires que j’installe sur chaque machine :

  • GitHub Desktop — un client Git visuel lorsque je ne souhaite pas utiliser l’interface en ligne de commande Docker Desktop — le conteneur runtime que j’utilise ; Podman Desktop constitue une alternative
  • Ollama ou LM Studio — exécuter LLMs localement sur le Mac

10. Voilà la configuration requise

C’est exactement ce que j’installe sur chaque nouvel MacBook avant de commencer à travailler réellement. Ajustez selon vos préférences.

Principaux enseignements

  1. Il s’agit d’une configuration personnelle, et non d’une base minimale ou universelle pour macOS. Supprimez tout ce que vous n’utilisez pas.
  2. Les éléments fournis avec /bin/zsh Cela est suffisant pour cette configuration. Installez uniquement la version Zsh fournie par Homebrew lorsque vous avez besoin d’une version plus récente spécifique issue du noyau original.
  3. Dans mon flux de travail quotidien, uv gère désormais l’installation de Python ainsi que les environnements de projet. pyenv reste optionnel pour les interpréteurs CPython compilés manuellement ou personnalisés.
  4. Documentez les commandes d’installation, les fichiers de configuration sans données sensibles, les extensions d’éditeur et les emplacements des modèles afin que la prochaine configuration se fasse de manière mécanique.
