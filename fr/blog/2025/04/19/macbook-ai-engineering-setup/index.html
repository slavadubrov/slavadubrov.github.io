Configuration du MacBook pour l’ingénierie IA : outils macOS, Python, Docker et flux de travail en terminal

Voici ma liste de contrôle pour transformer une installation macOS fraîche en poste de travail d’ingénierie IA : outils en ligne de commande, Homebrew, Python, Docker, paramètres par défaut du terminal, ainsi que la configuration de l’éditeur.

En résumé : Une configuration pratique d’un MacBook pour l’ingénierie IA est délibérément simple : Homebrew, uv, Docker ou Colima, Git, configuration de la shell, outils d’édition, utilitaires locaux pour les LLM/OCR, ainsi qu’une sauvegarde reproductible des fichiers de configuration. Il convient d’optimiser pour la vitesse de reconstruction, et non pour la nouveauté.

1. Installer les outils en ligne de commande Xcode

xcode-select --install

Cela ouvre une boîte de dialogue qui vous guide pas à pas lors de l’installation.

Homebrew c’est le gestionnaire de paquets que j’utilise pour tout le reste de cette liste :

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

L’installateur demande votre mot de passe et s’ajoute à votre PATH## 3. Installer les outils de base

brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

Groupés de manière approximative selon leur usage :

Environnement Python

  • pyenv — gérer plusieurs versions de Python en parallèle UV — Gestionnaire de paquets Python rapide et gestionnaire d’environnements

Bibliothèques système

  • openssl — Support de la cryptographie SSL/TLS lecture de ligne — édition de texte en ligne de commande sqlite3 — base de données embarquée légère xz — compression des données zlib — bibliothèque de compression

Outils de productivité

  • htop — moniteur du système visuel et visualiseur de processus tmux — Gérer plusieurs sessions de terminal ncdu — analyser l’utilisation du disque de manière interactive gitmoji — Ajouter des emojis aux messages de commit pandoc — Convertir des documents entre différents formats

Pour en savoir plus sur uv en soi, voir le mien Guide rapide pour la gestion de Python sous macOS à l’aide de uv## 4. Choisir un terminal

Le Terminal par défaut de macOS convient parfaitement. J’ai utilisé iTerm2 pendant des années, et a récemment déménagé à Déformation, un terminal basé sur Rust doté de fonctionnalités d’IA intégrées. Vous pouvez obtenir Warp directement auprès d’eux. site web### Optionnel : Configuration d’iTerm2

Si vous continuez à utiliser iTerm2, voici les deux paramètres que je modifie systématiquement :

Activer l’édition de texte naturelle

  1. Préférences → Profils → Clés → Mappages de clés
  2. Cliquez sur le menu déroulant « Préfixes… »
  3. Sélectionnez « Édition de texte naturelle »

Choisir un thème de couleur

  1. Parcourez les thèmes disponibles à iTerm2 – Thèmes de couleur
  2. Préférences → Profils → Couleurs → Préférences de couleur…
  3. Cliquez sur « Importer » et sélectionnez le thème que vous avez téléchargé

5. Configurer Zsh avec Oh My Zsh

macOS utilise par défaut Zsh. Oh mon Zsh ajoute des valeurs par défaut raisonnables ainsi qu’un système de plugins en complément :

brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

L’installateur fait une sauvegarde de votre configuration Zsh existante avant d’installer la nouvelle. ~/.zshrc pour charger ces plugins :

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Les descriptions de tous ceux-ci se trouvent dans le Oh mon wiki des plugins ZshLes deux derniers nécessitent une installation distincte :

  • zsh-autosuggestions — suggère des commandes au fur et à mesure que vous tapez, en se basant sur votre historique zsh-syntax-highlighting — Les commandes de couleur s’affichent au fur et à mesure que vous tapez, ce qui rend les fautes d’orthographe évidentes.

Suivez les instructions d’installation indiquées dans chaque répertoire.

7. Personnalisez le thème avec Powerlevel10k

Powerlevel10k C’est le thème Zsh que j’utilise. Il affiche le répertoire de travail, l’état Git et l’environnement Python actif dans la invite de commande, et il dispose d’un assistant interactif permettant d’ajuster l’apparence. Les instructions d’installation se trouvent sur le page GitHub.

Polices dans d’autres éditeurs

Si vous utilisez VSCode ou un autre éditeur doté d’une terminal intégrée, configurez la police de la terminal afin que les icônes Powerlevel10k s’affichent correctement :

  1. Ouvrez les paramètres de votre éditeur
  2. Recherchez terminal.integrated.fontFamily
  3. Définissez-le sur MesloLGS NF (installé en même temps que Powerlevel10k)

Le paramétrage complet de la police se trouve dans le Guide de police Powerlevel10k## 8. Éditeur et assistants IA

J’ai systématiquement un IDE ouvert, accompagné d’un ou deux outils IA en exécution à côté.

IDEs

  • Curseur — une branche de VSCode intégrant le pair programming assisté par l’IA VSCode — celui qui dispose d’un vaste catalogue d’extensions OpenAI Codex — génération et complétion de code via OpenAI Claude — L’assistant d’Anthropic ; mon outil de prédilection pour les tâches les plus complexes.

9. Le reste

Quelques outils supplémentaires que j’installe sur chaque machine :

  • GitHub Desktop — client Git visuel lorsque je ne souhaite pas utiliser l’interface en ligne de commande Docker — conteneurs ; si vous préférez ne pas l’utiliser, Podman et d’autres alternatives couvrir la majeure partie des mêmes sujets Ollama ou LM Studio — exécuter des LLM localement sur le Mac

10. Voilà la configuration

C’est ce que j’installe sur chaque nouvel MacBook avant de commencer à travailler réellement. Ajustez selon vos préférences.

Points clés

  1. Rendre la configuration reproductible : documenter les commandes d’installation, les fichiers dotfiles, les clés SSH, les extensions d’éditeur ainsi que les emplacements des modèles et du cache.
  2. Séparer la gestion des paquets système de la gestion des projets Python. Homebrew et uv résolvent des problèmes différents.
  3. Installer uniquement les outils d’IA locaux une fois que l’environnement de développement de base est stable.
  4. La qualité d’une configuration de poste de travail se mesure à la vitesse à laquelle on peut la réinstaller après une installation propre de macOS.
