    Configuração do MacBook para Engenharia de IA: Ferramentas do macOS, Python, Docker e Fluxo de Trabalho no Terminal

    Esta é a minha lista de verificação para transformar uma instalação fresca do macOS numa estação de trabalho para engenharia de IA: ferramentas de linha de comando, Homebrew, Python, Docker, configurações padrão do terminal e preparação do editor.

    TL;DR: Uma configuração prática de MacBook para engenharia de IA é intencionalmente simples: Homebrew, uv, Docker ou Colima, Git, configurações da shell, ferramentas de editoração, utilitários locais para LLM/OCR e um backup repetível dos ficheiros de configuração. Otimize para acelerar a reconstrução, não para novidade.

    1. Instalar as Ferramentas de Linha de Comando do Xcode

    xcode-select --install

    Isso abre uma caixa de diálogo que o orienta ao longo da instalação.

    Homebrew é o gestor de pacotes que utilizo para tudo o resto nesta lista:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    O instalador solicita a sua palavra-passe e adiciona-se ao seu PATH## 3. Instalar as ferramentas principais

    brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

    Agrupados de forma aproximada conforme a sua finalidade:

    Ambiente Python

    • pyenv — gerir múltiplas versões do Python em paralelo UV — gestor de pacotes Python rápido e manipulador de ambientes

    Bibliotecas do sistema

    • openssl — Suporte à criptografia SSL/TLS readline — edição de texto na linha de comandos sqlite3 — base de dados embutida leve xz — compressão de dados zlib — biblioteca de compressão

    Ferramentas de produtividade

    • htop — Monitor de sistema visual e visualizador de processos tmux — gerir múltiplas sessões de terminal ncdu — analisar o uso do disco de forma interativa gitmoji — adicionar emojis a mensagens de commit
    • pandoc — converter documentos entre formatos

    Para saber mais sobre uv em si mesmo, veja o meu Guia Rápido para gerir o Python no macOS com uv## 4. Escolher um terminal

    O Terminal padrão do macOS já é suficiente. Eu utilizei iTerm2 há anos e recentemente mudou-se para Warpação, um terminal baseado em Rust com funcionalidades de IA integradas. Pode obter o Warp diretamente na sua página oficial. site web### Opcional: Configuração do iTerm2

    Se optar por manter o iTerm2, estas são as duas configurações que eu altero sempre:

    Ativar a edição de texto natural

    1. Preferências → Perfis → Teclas → Mapeamentos de Teclas
    2. Clique na lista suspensa “Predefinições…”
    3. Selecione “Edição de Texto Natural”

    Escolher um tema de cor

    1. Navegue pelos temas em iTerm2 – Esquemas de Cores
    2. Preferências → Perfis → Cores → Predefinições de Cor…
    3. Clique em Importar e selecione o tema que transferiu

    5. Configurar o Zsh com Oh My Zsh

    O macOS utiliza o Zsh por predefinição. Oh meu Zsh adiciona valores padrão sensatos e um sistema de plugins em cima disso:

    brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

    O instalador faz um backup da sua configuração Zsh existente antes de substituí-la pela nova. ~/.zshrc para carregar estes plugins:

    plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

    As descrições para todos estes encontram-se em Oh My Zsh plugins wikiOs dois últimos exigem uma instalação separada:

    • zsh-autosuggestions — sugere comandos à medida que escreve, com base no seu histórico destaque-sintaxe-zsh — Colore os comandos à medida que escreve, de modo a que os erros de digitação fiquem evidentes.

    Siga as instruções de instalação em cada repositório.

    7. Personalize com o Powerlevel10k

    Powerlevel10k é o tema Zsh que utilizo. Ele exibe o diretório de trabalho, o estado do Git e o ambiente Python ativo no prompt, além de dispor de um assistente interativo para ajustar a aparência. As instruções de instalação encontram-se em página do GitHub### Fontes noutros editores

    Se utilizar o VSCode ou outro editor com um terminal integrado, defina a fonte do terminal para que os ícones do Powerlevel10k sejam exibidos corretamente:

    1. Abra as definições do seu editor
    2. Procure por terminal.integrated.fontFamily
    3. Defina-o para MesloLGS NF (instalado juntamente com o Powerlevel10k)

    Configuração completa da fonte encontra-se no Guia de fontes Powerlevel10k## 8. Editores e assistentes de IA

    Acabo sempre com um IDE aberto e uma ou duas ferramentas de IA a correr ao seu lado.

    IDEs

    • Cursor — um fork do VSCode com programação em parceria por IA integrada VSCode — aquele com um catálogo de extensões enorme

    Assistentes de IA

    • OpenAI Codex — geração e completação de código via OpenAI Claude — O assistente da Anthropic; o que recorro quando enfrento tarefas mais difíceis.

    Atualmente, é o Cursor com o Codex e o Claude Code a serem executados em paralelo.

    9. O resto

    Alguns complementos que instalo em cada máquina:

    • GitHub Desktop — cliente visual do Git quando não quero recorrer à linha de comandos Docker — containers; se preferir não utilizá‑lo, Podman e outras alternativas abordar a maior parte dos mesmos tópicos Ollama ou LM Studio — executar LLMs localmente no Mac

    10. Essa é a configuração

    É isto que instalo em cada novo MacBook antes de começar a trabalhar de verdade. Ajuste conforme preferir.

    Pontos Chave

    1. Torne a configuração reprodutível: documente os comandos de instalação, ficheiros de configuração, chaves SSH, extensões de editor e locais dos modelos/cache.
    2. Separe a gestão de pacotes do sistema da gestão de projetos em Python. O Homebrew e o uv resolvem problemas diferentes.
    3. Instale as ferramentas de IA locais apenas depois de o ambiente de desenvolvimento básico estar estável.
    4. Uma boa configuração para estação de trabalho é medida pela rapidez com que consegue recriá-la após uma instalação limpa do macOS.
