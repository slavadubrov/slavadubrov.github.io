Configuração do MacBook para Engenharia em AI: Ferramentas do macOS, Python, Docker e Fluxo de Trabalho no Terminal

Esta é a minha lista de verificação para transformar uma instalação fresca do macOS numa estação de trabalho de engenharia AI: ferramentas da linha de comandos, Homebrew, Python, Docker, definições padrão do terminal e configuração do editor.

TL;DR: Esta é a configuração que utilizo num novo MacBook: Homebrew, Python com uv, o Zsh incluído com Oh My Zsh e Powerlevel10k, Docker, os meus editores de código e as ferramentas locais AI. Mantenho os comandos e os ficheiros de configuração em controlo de versões para garantir que a configuração seja reprodutível.

1. Instalar as Ferramentas de Linha de Comando do Xcode

xcode-select --install

Isto abre uma caixa de diálogo que o orienta ao longo da instalação.

2. Instalar o Homebrew

Homebrew é o gestor de pacotes que utilizo para tudo o resto nesta lista:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

O instalador explica quais alterações serão feitas e solicita confirmação. Em Apple Silicon, o Homebrew utiliza /opt/homebrew por predefinição. Siga o brew shellenv O comando é exibido no final; o Homebrew não torna automaticamente esse comando disponível em todos os shells de forma silenciosa.

3. Instalar as ferramentas principais

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

Agrupados de forma aproximada de acordo com a sua finalidade:

Ambiente Python

Antes de adotar o uv, utilizava o pyenv como base para a gestão do meu ambiente Python. Atualmente, o uv cobre todo esse fluxo de trabalho diário por mim, incluindo a instalação do Python e a criação de ambientes de projeto.

  • UV instala versões do Python e gere ambientes de projeto, dependências, comandos e ficheiros de bloqueio. pyenv É opcional. Utilizo-o apenas quando preciso de um interpretador compilado a partir da fonte ou de opções de compilação personalizadas para o CPython que as distribuições pré-compiladas do uv não disponibilizam.

Instale o pyenv separadamente quando precisar desse caminho de compilação a partir da fonte:

brew install pyenv

Bibliotecas de sistema

  • openssl fornece suporte à criptografia SSL/TLS. leitura de linha adiciona funcionalidades de edição de texto na linha de comandos. sqlite3 É uma base de dados embebida de baixo consumo.
  • xz e zlib Fornecer compressão de dados.

Ferramentas de produtividade

  • htop é um monitor de sistema e processo visual. tmux gerencia múltiplas sessões de terminal. ncdu Analisa o uso do disco de forma interativa.
  • gitmoji adiciona emojis a mensagens de commit. pandoc converte documentos entre formatos.

Para saber mais sobre uv em si mesmo, consulte o meu Guia Rápido para gerir o Python no macOS com uv.

4. Escolher um terminal

O Terminal padrão do macOS é suficiente. Eu utilizei iTerm2 há anos e recentemente mudou-se para Warpação, um terminal baseado em Rust com funcionalidades integradas de AI. A escolha é pessoal; nenhuma das etapas seguintes de configuração depende dela.

Opcional: Configuração do iTerm2

Se continuar a usar o iTerm2, existem duas definições que eu altero sempre:

Ativar edição de texto natural

  1. Preferências → Perfis → Chaves → Mapeamentos de Chave
  2. Clique na lista suspensa Presets…
  3. Selecione “Edição de Texto Natural”

Escolher um tema de cor

  1. Navegue pelos temas em iTerm2-Esquemas-de-Cor
  2. Preferências → Perfis → Cores → Predefinições de Cor…
  3. Clique em Importar e selecione o tema que transferiu

5. Configurar o Zsh

O macOS já inclui o Zsh e utiliza‑o como shell de login padrão. Eu uso o que vem incluído. /bin/zsh; instalar Zsh do Homebrew apenas quando for necessário utilizar uma versão específica mais recente proveniente da cadeia de desenvolvimento principal.

Verifique o Zsh instalado e a shell de login atual:

echo "$SHELL"
command -v zsh
zsh --version

Se o Zsh estiver instalado mas não selecionado como shell de login, mude para a cópia incluída:

chsh -s /bin/zsh

Abra um novo terminal após alterar o shell de login.

Oh meu Zsh acrescenta os valores predefinidos e o sistema de plugins que utilizo por cima. O seu instalador oficial é:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

O projeto recomenda inspecionar um instalador desconhecido antes de o executar. Documenta também que um existente .zshrc é renomeado para .zshrc.pre-oh-my-zsh durante a instalação.

6. Adicionar plugins ao Zsh

Edite o seu ~/.zshrc para carregar estes plugins:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Remover pyenv da lista de plugins, caso tenha ignorado a instalação opcional do pyenv.

As descrições para todos estes encontram-se no Oh My Zsh plugins wiki.

Os dois últimos exigem uma instalação separada:

  • zsh-autosuggestions sugere comandos da sua história à medida que escreve. destaque-sintaxe-zsh Os comandos são coloridos à medida que se digita, o que facilita a deteção de erros de digitação.

Siga as instruções de instalação em cada repositório.

7. Defina o tema com Powerlevel10k

Powerlevel10k é o tema Zsh que utilizo. Ele exibe o diretório de trabalho, o estado do git e o ambiente Python ativo no prompt, e dispõe de um assistente interativo para ajustar a aparência. As instruções de instalação encontram‑se na página do GitHub.

Fontes noutros editores

Se utilizar o VSCode ou outro editor com um terminal integrado, defina a fonte do terminal de modo a que os ícones do Powerlevel10k sejam exibidos corretamente:

  1. Abra as definições do seu editor
  2. Procure por terminal.integrated.fontFamily
  3. Defina-o para MesloLGS NF (instalado em conjunto com o Powerlevel10k)

Configuração completa da fonte encontra-se no Guia de fontes Powerlevel10k.

É isso que instalo em cada novo MacBook antes de começar a trabalhar de verdade. Ajuste conforme preferir.

Principais conclusões

  1. Trata‑se de uma configuração pessoal, e não de um ponto de partida mínimo ou universal para o macOS. Remova tudo o que não utilizar.
  2. O pacote incluído /bin/zsh É suficiente para esta configuração. Instale apenas a versão Zsh do Homebrew quando precisar de uma versão mais recente específica disponível na comunidade de desenvolvimento.
  3. O uv passou a ser responsável pela instalação do Python e pelos ambientes de projeto no meu fluxo de trabalho diário. O pyenv continua a ser opcional para interpretadores CPython compilados manualmente ou personalizados.
  4. Mantenha documentados os comandos de instalação, os ficheiros de configuração sem dados sensíveis, as extensões do editor e as localizações dos modelos, para que a próxima configuração seja totalmente automatizada.
