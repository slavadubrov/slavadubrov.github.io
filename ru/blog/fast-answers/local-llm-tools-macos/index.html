Лучшие локальные инструменты для LLM под macOS

Инструменты локальных LLM на macOS делятся по назначению: простая обработка запросов, работа с интерфейсами пользователя, низкоуровневый контроль инференса и эксперименты, оптимизированные под Apple Silicon. Один инструмент не обязан выполнять все четыре функции.

Стандартная конфигурация: Ollama для создания локального API. LM Studio — для исследования моделей. llama.cpp — когда требуется контроль над средой выполнения формата GGUF. MLX — при необходимости работы с Python напрямую на чипах Apple Silicon рядом с самой моделью.

Какой из них следует установить в первую очередь?

Если вы разрабатываете программное обеспечение, сначала установите Ollama. Многие приложения поддерживают взаимодействие с ним, а локальный API вполне достаточен для прототипов, тестирования и небольших внутренних инструментов. Это самый короткий путь от мысли «Мне нужна локальная модель» до реализации, когда «мое приложение может вызывать локальную модель».

Сначала установите LM Studio, если вы собираетесь выбирать модель. Это инструмент, удобный для просмотра моделей, изменения настроек, сравнения результатов и запуска локального сервера, совместимого с OpenAI, без необходимости самостоятельной настройки рабочего процесса.

Установите llama.cpp в первую очередь, если вас интересует внутренняя работа процесса инференса. Длина контекста, квантизация, параметры Metal, обработка промптов, размеры пакетов данных и поведение сервера гораздо легче анализировать, когда вы находитесь непосредственно в момент выполнения кода.

Используйте MLX, когда вам нужно делать больше, чем просто предоставлять доступ к чат-модели. Он подходит для экспериментов с моделями, разработанными специально для Apple Silicon, их конвертации, финальной настройки, а также для рабочих процессов на Python, в которых единая память является частью архитектуры.

Примечания по аппаратному обеспечению

Единая память представляет собой настоящее ограничение для чипов Apple Silicon. Модель, которая помещается в MacBook Pro с 64 ГБ памяти, может оказаться непригодной для использования в MacBook Air с 8 ГБ памяти. Квантование помогает, однако длина контекста способна значительно потреблять память. Рекомендуется тестировать реальную структуру запроса, а не только название модели.

Для небольших локальных инструментов модели объёмом 7 млрд или 8 млрд параметров зачастую полезнее, чем перегруженные модели большего размера. В задачах программирования длина контекста и возможность интеграции с инструментами могут иметь большее значение, чем просто позиция в тестовых рейтингах. Для проверки качества документов приоритетом при выборе локальной модели обычно является качество поиска информации.

Чего не стоит делать

Не превращайте настройку локального LLM в постоянный проект для тестирования, если конечной целью не является повышение производительности. Начните с Ollama или LM Studio. Докажите, что локальная обработка данных действительно помогает. Только после этого, имея конкретные основания, переходите к использованию llama.cpp или MLX.

Не сравнивайте модели исключительно в интерфейсе чата, если реальная нагрузка связана со структурированным извлечением данных, редактированием кода или синтезом ответов с использованием технологии RAG. Напишите небольшой скрипт для оценки с типичными промптами.
