Учебник по серверу MCP с использованием uv и FastMCP: создание сервера FeatureStoreLite

В этом учебнике по серверам MCP мы создадим небольшой сервер хранилища признаков с помощью FastMCP, запустим его с помощью uv и подключим к Claude Desktop.

Кратко: Для предоставления агенту полезного локального инструмента достаточно небольшого сервера FastMCP. Используйте uv для воспроизводимой настройки, ограничьте объём сервера, валидируйте входные данные с помощью типизированных схем и протестируйте инструмент MCP перед подключением его к модели.

Что такое сервер MCP?

Model Context Protocol (MCP) — это открытый стандарт для подключения ИИ-ассистентов, таких как Claude, к внешним данным и инструментам. Один протокол, один набор конвенций вместо необходимости разрабатывать новую интеграцию для каждого инструмента, который вы хотите использовать.

В этом руководстве мы создадим сервер MCP FeatureStoreLite. Он располагается между LLM и хранилищем признаков (базой данных предварительно вычисленных ML-признаков) и предоставляет инструменты для запроса и записи векторов признаков, ключи к которым формируются на основе пользователя, продукта или документа.

Зачем это делать?

Вы — инженер машинного обучения, отлаживающий пайплайн. Вместо того чтобы обращаться к SQL или писать временный скрипт для проверки значений признаков, вы можете напрямую спросить у Claude: «Каков вектор признаков для пользователя_123?» или «Покажите метаданные для продукта_abc.»_ Именно сервер MCP делает это возможным.

Зачем использовать uv?

Мы будем использовать UV — это инсталлятор пакетов для Python и менеджер проектов, который работает во много раз быстрее pip и объединяет в одном инструменте решение проблем зависимостей и управление виртуальными средами. Другая причина: uv run --with mcp[cli] ... Давайте позже укажем зависимости непосредственно в конфигурации Claude Desktop, чтобы не приходилось поддерживать отдельную среду venv в синхронизации.

  1. Пользователь задаёт вопрос на естественном языке.
  2. Claude Desktop является клиентом MCP. Он выбирает соответствующий инструмент на основе вопроса и вызывает его.
  3. Наш FastMCP сервер предоставляет доступ к get_feature и store_feature в качестве инструментов MCP.
  4. SQLite служит основным хранилищем для векторов признаков.

Создайте новую директорию и инициализируйте проект на Python. uv init создаёт pyproject.toml для вас.

mkdir mcp-featurestore
cd mcp-featurestore

# Initialize Python project
uv init

# Add the MCP SDK with CLI tools
uv add "mcp[cli]"

3. Создание сервера

Два файла, разделённых по функциональным областям:

  1. database.py обрабатывает операции с SQLite. featurestore_server.py определяет сервер MCP.

3.1. Слой базы данных (database.pyЭтот модуль управляет соединением с SQLite и содержит несколько вспомогательных функций. В него загружаются две примерные строки, чтобы сервер имел что вернуть при первом запросе. database.py:

# database.py
import json
import os
import sqlite3


def get_db_path() -> str:
    """Get the database path - always in the script's directory"""
    script_dir = os.path.dirname(os.path.abspath(__file__))
    return os.path.join(script_dir, "features.db")


def init_db() -> None:
    """Initialize the feature store database with table and sample data"""
    conn = sqlite3.connect(get_db_path())
    conn.execute("""
        CREATE TABLE IF NOT EXISTS features (
            key TEXT PRIMARY KEY,
            vector TEXT NOT NULL,
            metadata TEXT,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    """)

    # Sample data for experimentation
    example_features = [
        (
            "user_123",
            "[0.1, 0.2, -0.5, 0.8, 0.3, -0.1, 0.9, -0.4]",
            json.dumps({"type": "user", "id": 123, "segment": "premium"}),
        ),
        (
            "product_abc",
            "[0.7, -0.3, 0.4, 0.1, -0.8, 0.6, 0.2, -0.5]",
            json.dumps({"type": "product", "id": "abc", "category": "electronics"}),
        ),
    ]

    # Insert if not exists
    for key, vector, metadata in example_features:
        try:
            conn.execute(
                "INSERT INTO features (key, vector, metadata) VALUES (?, ?, ?)",
                (key, vector, metadata),
            )
        except sqlite3.IntegrityError:
            pass  # Already exists

    conn.commit()
    conn.close()


def get_db_connection() -> sqlite3.Connection:
    """Get a database connection"""
    return sqlite3.connect(get_db_path())


if __name__ == "__main__":
    init_db()
    print("✅ Database initialized successfully!")

Инициализация базы данных:

uv run python database.py

3.2. Сервер MCPfeaturestore_server.py)

FastMCP Выполняет основную часть работы. Достаточно добавить декоратор к обычной функции на Python, и она будет зарегистрирована как инструмент или ресурс MCP. Документация функции становится описанием, которое видит большая языковая модель, поэтому её следует писать с учётом потребностей модели, а не только для людей. featurestore_server.py:

# featurestore_server.py
import json
from mcp.server.fastmcp import FastMCP
from database import get_db_connection, init_db

# Initialize the MCP Server
mcp = FastMCP("FeatureStoreLite")

# Ensure DB is ready when server starts
init_db()


@mcp.resource("schema://main")
def get_schema() -> str:
    """
    Resource: Provide the database schema.
    Resources are passive data that LLMs can read like files.
    """
    conn = get_db_connection()
    try:
        schema = conn.execute(
            "SELECT sql FROM sqlite_master WHERE type='table'"
        ).fetchall()
        return "\n".join(sql[0] for sql in schema if sql[0]) or "No tables found."
    finally:
        conn.close()


@mcp.tool()
def store_feature(key: str, vector: str, metadata: str | None = None) -> str:
    """
    Tool: Store a feature vector.
    Tools are executable functions that LLMs can call to perform actions.
    """
    conn = get_db_connection()
    try:
        # Validate that vector is valid JSON
        json.loads(vector)

        conn.execute(
            "INSERT OR REPLACE INTO features (key, vector, metadata) VALUES (?, ?, ?)",
            (key, vector, metadata),
        )
        conn.commit()
        return f"Successfully stored feature '{key}'"
    except json.JSONDecodeError:
        return "Error: Vector must be a valid JSON array string (e.g., '[0.1, 0.2]')"
    except Exception as e:
        return f"Error: {str(e)}"
    finally:
        conn.close()


@mcp.tool()
def get_feature(key: str) -> str:
    """
    Tool: Retrieve a feature vector by key.
    """
    conn = get_db_connection()
    try:
        row = conn.execute(
            "SELECT vector, metadata FROM features WHERE key = ?", (key,)
        ).fetchone()

        if row:
            return json.dumps(
                {
                    "key": key,
                    "vector": json.loads(row[0]),
                    "metadata": json.loads(row[1]) if row[1] else None,
                },
                indent=2,
            )
        return f"Feature '{key}' not found."
    finally:
        conn.close()


@mcp.tool()
def list_features() -> str:
    """
    Tool: List all available feature keys.
    """
    conn = get_db_connection()
    try:
        rows = conn.execute("SELECT key FROM features").fetchall()
        return json.dumps([row[0] for row in rows])
    finally:
        conn.close()


if __name__ == "__main__":
    mcp.run()

4. Тестирование с MCP Inspector

Прежде чем подключать это к Claude, проведите проверку работоспособности сервера с помощью MCP Inspector. Это небольшой веб-интерфейс для прямого вызова инструментов и чтения ресурсов.

uv run mcp dev featurestore_server.py

Эта команда запускает сервер и открывает инспектор в вашем браузере (обычно по адресу http://localhost:5173).

5. Подключение к Claude Desktop

Как только инспектор подтвердит, что сервер функционирует, зарегистрируйте его в Claude Desktop.

5.1. Настройка Claude

Отредактируйте файл конфигурации Claude Desktop:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%/Claude/claude_desktop_config.json

добавьте свой сервер в mcpServers объект:

{
    "mcpServers": {
        "featurestore": {
            "command": "uv",
            "args": [
                "run",
                "--with",
                "mcp[cli]",
                "mcp",
                "run",
                "/ABSOLUTE/PATH/TO/mcp-featurestore/featurestore_server.py"
            ]
        }
    }
}

Важно: путь к featurestore_server.py должен быть абсолютным. Относительный путь приведёт к тихой ошибке, поскольку Claude Desktop запускается из другого рабочего каталога.

5.2. Как происходит взаимодействие

Вот что происходит от начала до конца, когда Claude нужно найти определение функции:

  1. Claude видит доступные инструменты (get_feature, list_features, и так далее).
  2. Он определяет, что для решения задачи требуются данные из хранилища признаков.
  3. Он формирует запрос к инструменту и отправляет его на ваш сервер.
  4. Сервер запускает функцию на Python и возвращает результат.
  5. Claude использует этот результат для формирования окончательного ответа.

5.3. Примеры запросов

Перезапустите Claude Desktop и попробуйте несколько вводных команд:

  1. «Перечислите все доступные функции». {width=“600”}

  2. «Получить вектор признаков для user_123».} {width=“600”}

6. Устранение неполадок

Несколько типов сбоев, о которых стоит знать:

  • «Не удалось установить соединение с сервером»:

    • Проверьте журналы логов. ~/Library/Logs/Claude/mcp.log на macOS.
    • Убедитесь, что в конфигурации используется абсолютный путь, а не относительный.
    • Убедитесь uv входит в состав Claude Desktop PATH. Если это не так, укажите полный двоичный путь (which uv это укажет местоположение файла).

  • «Ошибка выполнения инструмента»:

    • Воспроизведите её в Инспекторе с uv run mcp dev featurestore_server.py. Инспектор отображает исходную ошибку, которую Claude Desktop обычно скрывает.
    • Проверьте это. features.db создаётся рядом с database.py. Если изменится рабочая директория, относительный путь к скрипту будет get_db_path() это и есть то, что спасает вас.

7. Заключение

Вот и всё: а FastMCP сервер, хранилище на основе SQLite и конфигурация Claude Desktop, указывающая на него uv run команда. Тот же принцип применим ко всему, что можно обернуть в функцию на Python. Замените вызовы SQLite на настоящий хранилище функций, внутренний API или реестр моделей — сервер останется компактным.

Основные выводы

  1. MCP представляет собой границу инструментов, а не основание для открытия доступа к каждой внутренней функции.
  2. Сохраняйте инструменты сервера компактными, с типизацией и такими, которые легко тестировать без использования LLM.
  3. Используйте uv, чтобы среду для обучения можно было воссоздавать заново.
  4. Как только агент сможет вызывать сервер MCP, рассматривайте его как код, предназначенный для производственной среды.
