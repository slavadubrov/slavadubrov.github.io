Настройка MacBook для инженерии в области ИИ: инструменты macOS, Python, Docker и рабочий процесс в терминале

Это мой чек-лист для превращения свежей установки macOS в рабочую станцию для инженерии в области ИИ: инструменты командной строки, Homebrew, Python, Docker, настройки терминала и настройка редактора.

Кратко: Практическая конфигурация MacBook для работы с ИИ предполагает минимализм — Homebrew, uv, Docker или Colima, Git, настройки оболочки, инструменты для редактора, локальные утилиты для LLM/OCR, а также регулярное резервное копирование файлов с расширением .dot. Оптимизируйте настройки под скорость пересборки проектов, а не под новизну.

1. Установка инструментов командной строки Xcode

xcode-select --install

Это откроет диалоговое окно, которое покажет вам пошаговую инструкцию по установке.

Homebrew это менеджер пакетов, который я использую для всего остального в этом списке:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

инсталлятор запрашивает у вас пароль и добавляет себя в ваше PATH## 3. Установка основных инструментов

brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

Примерная группировка по назначению:

Среда Python

  • pyenv — управление несколькими версиями Python параллельно UV — быстрый менеджер пакетов Python и инструмент для управления средами

  • openssl — Поддержка криптографии SSL/TLS функция readline — редактирование текста в командной строке sqlite3 — легковесная встраиваемая база данных xz — сжатие данных zlib — библиотека сжатия

Инструменты для повышения производительности

  • htop — монитор визуальной системы и просмотрщик процессов tmux — управление несколькими сессиями терминала ncdu — интерактивный анализ использования диска gitmoji — добавить эмодзи в сообщения о коммитах pandoc — конвертация документов между форматами

Для получения дополнительной информации uv само по себе, см. мой Краткое руководство по управлению Python на macOS с использованием uv## 4. Выбор терминала

Стандартный Terminal в macOS подойдёт без проблем. Я использовал iTerm2 на протяжении многих лет, а недавно переехал в Искривлениетерминал на базе Rust с встроенными функциями ИИ. Вы можете скачать Warp с их официального сайта веб-сайт### Опционально: настройка iTerm2

Если вы продолжаете использовать iTerm2, вот две настройки, которые я всегда изменяю:

Включение режима естественной обработки текста

  1. Preferences → Profiles → Keys → Key Mappings
  2. Нажмите на выпадающий список Presets…
  3. Выберите опцию «Natural Text Editing»

Выбор темы цветов

  1. Просмотреть темы можно на iTerm2-Схемы цветов
  2. Предпочтения → Профили → Цвета → Заранее настроенные цвета…
  3. Нажмите «Импортировать» и выберите скачанный тему

5. Настройка Zsh с использованием Oh My Zsh

По умолчанию в macOS используется Zsh. О, мой Zsh! добавляет разумные значения по умолчанию и систему плагинов сверху:

brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

Установщик создаёт резервную копию вашей существующей конфигурации Zsh перед заменой её на новую.

6. Добавление плагинов Zsh

Отредактируйте ваш файл ~/.zshrc чтобы загрузить эти плагины:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Описания для всех этих элементов приведены ниже. Вики плагинов Zsh. Последние два компонента требуют отдельной установки:

  • zsh-autosuggestions — предлагает команды по мере ввода на основе вашей истории заставка синтаксиса zsh — цвета команд отображаются в реальном времени по мере ввода, что позволяет сразу замечать опечатки.

Следуйте инструкциям по установке, приведённым в каждом репозитории.

7. Настройка темы с помощью Powerlevel10k

Уровень мощности 10к Это тема для Zsh, которую я использую. Она отображает рабочую директорию, статус Git и активную среду Python в промпте, а также предоставляет интерактивный мастер для настройки внешнего вида. Инструкции по установке находятся на страница GitHub### Шрифты в других редакторах

Если вы используете VSCode или другой редактор с встроенной терминалом, установите шрифт для терминала таким образом, чтобы иконки Powerlevel10k корректно отображались:

  1. Откройте настройки вашего редактора
  2. Поищите terminal.integrated.fontFamily
  3. Установите его на MesloLGS NF (установлено вместе с Powerlevel10k)

полном настроении шрифтов Руководство по шрифтам Powerlevel10k## 8. Редакторы и ИИ-ассистенты

У меня всегда открыт один IDE, рядом с которым работает один или два ИИ-инструмента.

IDE-ы Курсор — форк VSCode с встроенным функционалом парного программирования с использованием ИИ VSCode — тот, у которого огромный каталог расширений OpenAI Codex — генерация кода и автодополнение от OpenAI Клод — Ассистент от Anthropic; инструмент, к которому я обращаюсь при решении более сложных задач.

В наши дни это Cursor с Codex и Claude Code, работающие параллельно.

9. Остальное

Несколько дополнительных программ, которые я устанавливаю на каждом компьютере:

  • GitHub Desktop — визуальный клиент Git для случаев, когда не хочется пользоваться командной строкой Docker — контейнеры; если вы предпочитаете не использовать их, Podman и другие альтернативы покрывает практически ту же область Ollama или LM Studio — запускать LLM локально на Mac

Именно это я устанавливаю на каждом новом MacBook перед началом реальной работы. Пристраивайте всё по своему вкусу.

Основные выводы

  1. Сделайте процесс настройки воспроизводимым: задокументируйте команды установки, файлы конфигурации, ключи SSH, расширения редактора, а также пути к моделям и кэшу.
  2. Разделяйте управление системными пакетами и управление Python-проектами. Homebrew и uv решают разные задачи.
  3. Устанавливайте локальные инструменты для работы с ИИ только после того, как базовая среда разработчика станет стабильной.
  4. Хорошая настройка рабочей станции оценивается по скорости её воссоздания после чистой установки macOS.
