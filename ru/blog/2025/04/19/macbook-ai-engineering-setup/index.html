Настройка MacBook для AI-инжиниринг: инструменты macOS, Python, Docker и рабочий процесс в Терминале

Вот мой чек-лист для преобразования свежей установки macOS в рабочую станцию AI-инжиниринг: инструменты командной строки, Homebrew, Python, Docker, настройки терминала и настройка редактора.

Кратко: вот конфигурация, которую я использую на новом MacBook: Homebrew, Python с инструментом uv, встроенный Zsh с расширениями Oh My Zsh и Powerlevel10k, Docker, мои редакторы кода, а также локальные инструменты AI. Команды и файлы конфигурации хранятся под контролем версий, что обеспечивает возможность воспроизведения этой настройки в любое время.

1. Установка инструментов командной строки Xcode

xcode-select --install

Это открывает диалоговое окно, которое пошагово руководит вами в процессе установки.

2. Установка Homebrew

Homebrew это менеджер пакетов, который я использую для всего остального в этом списке:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Инсталлятор объясняет, что именно будет изменено, и запрашивает подтверждение. На чипах Apple Silicon Homebrew использует /opt/homebrew по умолчанию. Следуйте brew shellenv Команда выводится в конце; Homebrew не автоматически делает эту команду доступной во всех оболочках без явного указания.

3. Установка основных инструментов

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

Примерно сгруппировано по их назначению:

Среда Python

До внедрения uv я использовал pyenv в качестве основы для управления средой Python. Теперь uv полностью покрывает всю мою повседневную работу в этой области, включая установку Python и настройку сред для проектов.

  • UV устанавливает версии Python и управляет средами проектов, зависимостями, командами и файлами блокировки. pyenv Это необязательно. Я использую его лишь тогда, когда мне требуется интерпретатор, собранный из исходников, или специфические опции сборки CPython, которые отсутствуют в готовых дистрибутивах uv.

Установите pyenv отдельно, когда вам понадобится такой путь сборки из исходного кода:

brew install pyenv

Системные библиотеки

  • openssl поддерживает криптографию SSL/TLS. функция ввода строки добавляет возможность редактирования текста в командной строке. sqlite3 это легковесная встраиваемая база данных.
  • xz и zlib обеспечить сжатие данных.

Инструменты для повышения производительности

  • htop это монитор визуальной системы и процессов. tmux управляет несколькими терминалами сессии. ncdu анализирует использование диска в интерактивном режиме.
  • гитмоджи добавляет эмодзи в сообщения о коммитах. pandoc переводит документы между форматами.

Для получения дополнительной информации uv само по себе, см. мой Краткое руководство по управлению Python на macOS с использованием uv.

4. Выбор терминала

Стандартный Terminal в macOS полностью подходит. Я использовал iTerm2 на протяжении многих лет, а недавно переехал в Искривление пространства-времени, терминал на базе Rust с встроенными функциями AI. Выбор остаётся за пользователем; ни один из последующих этапов настройки не зависит от него.

Опционально: настройка iTerm2

Если вы остаётесь с iTerm2, вот два настройки, которые я всегда изменяю:

Включить редактирование естественного текста

  1. Предпочтения → Профили → Ключи → Сопоставления ключей
  2. Нажмите выпадающий список «Заданные настройки…»
  3. Выберите опцию «Редактирование текста в естественном формате»

Выберите тему цветов

  1. Просмотр тем iTerm2-шаблоны цветов
  2. Предпочтения → Профили → Цвета → Заранее настроенные цветовые схемы…
  3. Нажмите «Импортировать» и выберите загруженную тему.

5. Настройка Zsh

macOS уже включает Zsh и использует его в качестве стандартного оболочки для входа. Я использую поставляемую. /bin/zsh; установка Zsh от Homebrew только в том случае, если требуется конкретная более новая версия из исходного проекта.

Проверьте установленную версию Zsh и текущую оболочку входа:

echo "$SHELL"
command -v zsh
zsh --version

Если Zsh установлен, но не выбран в качестве оболочки для входа, переключитесь на встроенную копию:

chsh -s /bin/zsh

Откройте новую терминал после изменения оболочки входа.

О, мой Zsh Добавляет на базу используемые мной стандартные настройки и систему плагинов. Его официальный установщик — это:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Проект рекомендует ознакомиться с неизвестным установщиком перед его запуском. В нём также указано, что существующий .zshrc переименовывается в .zshrc.pre-oh-my-zsh во время установки.

6. Добавление плагинов Zsh

измените свой ~/.zshrc чтобы загрузить эти плагины:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

удалить pyenv из списка плагинов, если вы пропустили установку pyenv в качестве необязательного компонента.

Описания для всех этих элементов находятся в Вики плагинов Oh My Zsh.

Последние два компонента требуют отдельной установки:

  • заш-автопредложения предлагает команды из вашего исторического запися во время ввода текста. заширение синтаксиса zsh При вводе команды сразу отображаются соответствующие цвета, что позволяет быстрее обнаруживать опечатки.

Следуйте инструкциям по установке, приведённым в каждом репозитории.

7. Настройка темы с помощью Powerlevel10k

Powerlevel10k Это тема для Zsh, которую я использую. Она отображает рабочую директорию, статус Git и активную среду Python в промпт, а также предоставляет интерактивный мастер для настройки внешнего вида. Инструкции по установке находятся на страница GitHub.

Шрифты в других редакторах

Если вы используете VSCode или другой редактор с встроенной терминалом, настройте шрифт терминала таким образом, чтобы иконки Powerlevel10k корректно отображались:

  1. Откройте настройки вашего редактора.
  2. Выполните поиск. terminal.integrated.fontFamily
  3. Установите его на MesloLGS NF (установлено вместе с Powerlevel10k)

полном наборе шрифтов Руководство по шрифтам Powerlevel10k.

8. Редактор и помощники AI

В итоге у меня всегда открыт один средство разработки, а рядом с ним работает один-два инструмента AI.

IDE-инструменты

  • курсор — VSCode с встроенной поддержкой парного программирования AI через форк VSCode — тот, у которого огромный каталог расширений

ассистенты AI

  • OpenAI Codex — Агент для программирования от OpenAI Клод — Ассистент Anthropic; инструмент, к которому я обращаюсь при решении более сложных задач

В настоящее время это Cursor с Codex и Claude Code, работающие параллельно.

9. Остальное

Несколько дополнительных компонентов, которые я устанавливаю на каждой машине:

  • GitHub Desktop — визуальный клиент Git для случаев, когда не хочется пользоваться командной строкой Docker Desktop — контейнер рантайм, который я использую; Podman Desktop является альтернативой
  • Ollama или LM Studio — запустите LLMs локально на Mac.

10. Вот такая конфигурация

Именно это я устанавливаю на каждый новый MacBook перед началом серьёзной работы. Можете настраивать под собственные предпочтения.

Основные выводы

  1. Это личная конфигурация, а не минимальная или универсальная база macOS. Удалите всё, что вам не нужно.
  2. Входящие компоненты /bin/zsh Этого достаточно для данной конфигурации. Устанавливайте Zsh от Homebrew лишь тогда, когда вам требуется конкретная более новая версия из исходного репозитория.
  3. В моей повседневной работе управление установкой Python и средами проектов теперь осуществляется с помощью uv; pyenv остаётся факультативным инструментом для компилируемых или пользовательских интерпретаторов CPython.
  4. Обязательно документируйте команды установки, файлы конфигурации без содержащих секреты данных элементов, расширения редактора и пути модель, чтобы последующая настройка проходила автоматически.
