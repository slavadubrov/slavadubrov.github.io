Archivos de inicio de Zsh: ~/.zprofile versus ~/.zshrc en macOS y Linux

Si tu terminal parece lenta, o las variables de entorno no se cargan donde esperas, es probable que estés tropezando con el orden de inicio de Zsh.

La división entre ~/.zprofile y ~/.zshrc Es una de las fuentes más comunes de confusión al pasar a Zsh, especialmente en macOS, donde los valores predeterminados se comportan de forma distinta a los de Linux.

TL;DR

~/.zprofile Se utiliza para la configuración del entorno. Se ejecuta una vez por inicio de sesión, lo que en macOS implica una vez por pestaña de terminal. Coloque su PATH, EDITOR, y gestores de versiones como fnm o pyenv Ya está.

~/.zshrc Se utiliza para la configuración interactiva. Se ejecuta cada vez que se inicia un nuevo shell. Coloca allí alias, temas de prompt y asignaciones de teclas.

El flujo de inicio del shell

Para saber dónde colocar los elementos, es necesario conocer cuándo se cargan los archivos. Zsh dispone de una jerarquía específica al respecto.

Shells de inicio de sesión vs. shells interactivos

Un shell de inicio de sesión es el primer shell que se obtiene tras la autenticación. En macOS, por defecto, cada nueva pestaña o ventana de terminal es un shell de inicio de sesión. En Linux, abrir una terminal suele iniciar en su lugar un shell interactivo no de inicio de sesión, y de ahí proviene la mayor parte de la confusión entre plataformas.

Un shell interactivo es cualquier entorno en el que se pueden escribir comandos.

Así es como ocurre realmente al abrir una terminal en macOS:

El orden de carga

  1. ~/.zshenv (Opcional). Se ejecuta en cada shell, incluidos los scripts no interactivos. No coloques aquí salida ni lógica compleja, ya que esto podría dañar los scripts que incluyen como referencia tu configuración. Úsalo únicamente para las variables de entorno que deben existir en todas partes, lo cual es algo poco común para la mayoría de las personas. ~/.zprofile. Solo se ejecuta en shells de inicio de sesión. Tu fase de configuración.
  2. ~/.zshrc. Se ejecuta en shells interactivos. Su fase de personalización.
  3. ~/.zlogin (Opcional). Se ejecuta al final de la inicialización de una shell de inicio de sesión.

¿Qué va dónde?

~/.zprofile: la capa de entorno

Aquí es donde se configuran los elementos que heredan todos los demás procesos: rutas, variables y gestores de versiones del lenguaje.

Qué pertenece aquí:

  • PATH Modificaciones.
  • Variables de entorno: EDITOR, LANG, GOPATH, JAVA_HOME.
  • Inicialización de la herramienta que afecta al entorno: pyenv, rbenv, fnm, cargoEstos solo necesitan calcularse una vez. Si los incluyes en .zshrc, se vuelven a calcular cada vez que se abre un subshell o se ejecuta un script, lo cual no solo desperdicia tiempo sino que también puede generar entradas duplicadas en tu PATH.
# ~/.zprofile

# 1. Set up your PATH
# Local bin first so your tools override system ones
export PATH="$HOME/.local/bin:/opt/homebrew/bin:$PATH"

# 2. Global variables
export EDITOR="nvim"
export VISUAL="nvim"
export LANG="en_US.UTF-8"

# 3. Version managers (the heavy ones)
# Doing this here keeps shell startup fast
eval "$(fnm env --use-on-cd)"
eval "$(pyenv init -)"

~/.zshrc: la capa interactiva

Aquí es donde personalizas la shell en la que realmente escribes.

Qué debe ir aquí:

  • Alias: alias g='git'.
  • Tu prompt: Starship, Powerlevel10k, Pure.
  • Completaciones: compinit.
  • Vinculaciones de teclas: bindkey.
  • Opciones de shell: setopt autocd, setopt histignorealldupsEstos parámetros solo son relevantes cuando hay un humano frente al teclado. Un script que se ejecuta en segundo plano no necesita tu comando ni tus alias de Git.
# ~/.zshrc

# 1. Prompt
autoload -Uz promptinit && promptinit
prompt pure

# 2. Aliases
alias ll='ls -lah'
alias g='git'
alias gs='git status'

# 3. Shell options
setopt autocd              # cd by typing the directory name
setopt histignorealldups   # skip duplicate history entries
setopt share_history       # share history across tabs

# 4. Completions
autoload -Uz compinit && compinit

Quizás te estés preguntando: si .zprofile Se ejecuta primero; ¿por qué no definir allí sus alias y saltarse ese paso? .zshrc ¿Completamente?

Se trata, en esencia, de la diferencia entre herencia y redefinición.

Las variables de entorno se heredan

Cuando usted export EDITOR="vim" En un shell padre, todos los procesos hijos lo heredan: subshells, scripts y programas. Se establece una vez y se propaga por toda la estructura jerárquica, y por eso… .zprofile ¿Es el lugar adecuado para? export### Los alias y las funciones no lo hacen

Los alias como alias g='git' Las funciones de shell son locales al shell actual. No se transmiten a los shells hijos.

Si defines un alias en .zprofile, existe en tu shell de inicio de sesión de nivel superior. En el momento en que escribas zsh Para iniciar un subshell o ejecutar un script, ese alias ya no existe. Para que los alias estén disponibles en todas partes, es necesario redefinirlos en cada shell interactivo nuevo. Eso es exactamente lo que .zshrc Está destinado a…

Los scripts no necesitan funcionalidades propias de los seres humanos

Cuando ejecutas un script de shell (./deploy.sh), se inicia un nuevo shell no interactivo. No necesita tu prompt, no necesita tus alias de git, y ciertamente no quiere esperar a oh-my-zsh Para finalizar la carga. Se mantiene la configuración interactiva en .zshrc Esto significa que los scripts se ejecutan de forma rápida y limpia, sin que se filtre ninguna personalización propia.

Errores comunes

Implementación nvm o pyenv en .zshrc

Abres una nueva pestaña de terminal y pasan dos o tres segundos antes de poder escribir algo. Los gestores de versiones suelen tener un proceso de inicialización bastante pesado. .zshrc se ejecuta cada vez. Muévelos a ~/.zprofile Y la demora desaparece.

Un crecimiento PATH

El tuyo $PATH termina con los mismos directorios enumerados cinco veces. La causa es casi siempre export PATH="$HOME/bin:$PATH" se encuentra en .zshrc: en cada recarga (source ~/.zshrc) o el subshell vuelve a añadir la ruta. Mover PATH definiciones para ~/.zprofile### Volver a cargar tras realizar cambios

Cambios en ~/.zprofile No se aplica a su shell actual, porque .zprofile Se lee únicamente al iniciar sesión. Puede cerrar la pestaña e abrir una nueva (la opción más sencilla) o ejecutar source ~/.zprofile de forma manual. .zshrc, simplemente:

source ~/.zshrc

Una configuración que funciona en macOS y Linux

Si alternas entre diferentes máquinas (macOS en el trabajo, Linux en casa, o al revés), te encontrarás con un problema. Las terminales de Linux suelen iniciarse como shells sin conexión de usuario, lo que implica que omiten ~/.zprofile por completo.

La solución habitual es obtener los recursos desde .zprofile desde .zshrc cuando aún no se ha cargado:

# ~/.zshrc

# On Linux/non-login shells, make sure the environment is set
if [[ -o interactive && ! -o login ]]; then
    [[ -f ~/.zprofile ]] && source ~/.zprofile
fi

# ... rest of your interactive config

Resumen

ArchivoPropósitoEjemplos
~/.zshenvVariables de entorno críticasZDOTDIR (Solo para usuarios avanzados)
~/.zprofileConfiguración del entornoPATH, EDITOR, eval "$(pyenv init -)"
~/.zshrcConfiguración interactivaalias, prompt, bindkey, compinit

Dos reglas cubren la mayor parte de este caso. Las variables se heredan a lo largo del árbol de procesos, por lo que export pertenece a .zprofile. Los alias y las funciones no heredan, por lo que deben encontrarse en .zshrc Y deben ser redefinidos para cada nueva shell interactiva. Si las nuevas pestañas parecen lentas, el culpable es casi siempre una carga excesiva. pyenv init o nvm.sh se encuentra en .zshrc en lugar de .zprofile.

Conclusiones clave

  1. Utilizar .zprofile para la configuración de la shell de inicio de sesión, como la inicialización de PATH en Terminal de macOS.
  2. Utilizar .zshrc para el comportamiento de la shell interactiva, como los alias, la configuración del prompt, las completaciones y las opciones de shell.
  3. No duplique las ediciones de PATH en ambos archivos; la duplicación dificulta enormemente la depuración del orden de inicio.
  4. Si tiene dudas, coloque la inicialización del entorno en .zprofile y comodidades interactivas en .zshrc.
