Stable Diffusion en macOS: Dibujar elementos, DiffusionBee, ComfyUI, A1111 y Fooocus

Una aplicación de tipo arrastrar y soltar permite comenzar a trabajar de forma rápida; en cambio, el grafo de nodos requiere más tiempo para ser dominado, pero ofrece un acceso más amplio a pipeline. En Apple Silicon, la aplicación, la familia de modelos, la precisión, el tamaño de la imagen y el flujo de trabajo influyen directamente en si la generación cabe dentro de los límites establecidos. Elija la interfaz adecuada según el trabajo que necesite reproducir, y luego pruebe el modelo y las configuraciones específicas correspondientes.

TL;DR. Empiece con Draw Things o DiffusionBee cuando una aplicación nativa ya cumple con la tarea. Utilice ComfyUI cuando sea necesario inspeccionar, compartir o automatizar el grafo de procesamiento. Elija AUTOMATIC1111 únicamente si su flujo de trabajo depende de su interfaz o extensiones. Considere Fooocus como un flujo de trabajo estable para la era SDXL, y no como una vía para obtener soporte para modelos actuales: su proyecto oficial cuenta con un soporte a largo plazo limitado.

Un archivo de modelo no constituye un flujo de trabajo portátil

Varias aplicaciones pueden cargar .safetensors Los archivos, pero la existencia de una extensión compatible no garantiza la compatibilidad. Un pipeline completo también puede requerir una arquitectura específica, un codificador de texto, un VAE, un planificador, ControlNet, LoRA, un tokenizador o una configuración adaptada a la aplicación.

Registra la unidad de lanzamiento que generó una imagen:

  • aplicación y versión
  • repositorio de modelos, revisión y licencia
  • cada componente y adaptador del modelo
  • prompt, prompt negativo, semilla, muestreador, pasos, orientación y dimensiones
  • archivo de grafo o flujo de trabajo cuando la aplicación lo expone

Este registro es más importante que la popularidad de la aplicación. La misma semilla puede seguir generando una imagen distinta tras realizar cambios en el backend, el modelo, el muestreador o la precisión.

La memoria unificada es compartida, no ilimitada

El Apple Silicon permite que el CPU y el GPU accedan al mismo pool de memoria. Esto elimina la necesidad de un límite de copia independiente, pero macOS y todas las aplicaciones en ejecución compiten por esa misma capacidad.

En la generación de imágenes, el uso máximo implica más de checkpoint pesos:

peak memory ≈ model components
            + intermediate activations
            + attention and runtime workspace
            + loaded adapters and control models
            + application and OS memory

La resolución, el tamaño del lote, la arquitectura del modelo, la precisión, el escalado ascendente, ControlNet y el hecho de que los componentes permanezcan cargados influyen todos en el resultado. No compre ni seleccione una máquina basándose únicamente en una tabla genérica con especificaciones como “8/16/32 GB”. Pruebe el flujo de trabajo previsto y observe la presión sobre la memoria. Una generación que solo es posible gracias al uso continuo del intercambio de memoria no constituye un punto de operación óptimo.

Dibujar elementos: interacción nativa y scripting

Dibujar elementos Se trata de una aplicación para la plataforma Apple destinada a la generación local de imágenes. Su documentación actual aborda los modelos, las LoRAs, ControlNet, la inversión textual, Core ML, y la programación con scripts en JavaScript versionados.

Elige esta opción cuando el trabajo debe permanecer dentro de una aplicación nativa, pero se requiere un nivel de control superior al que ofrece un cuadro prompt. Antes de adoptarla de forma generalizada, importa los componentes del modelo exactos y reproduce un flujo de trabajo de edición, adaptación o control representativo. Por “nativa” se entiende la interfaz e implementación; no se garantiza que sea compatible ni que ofrezca las mismas velocidades en todas las arquitecturas nuevas.

DiffusionBee: flujo de trabajo empaquetado para macOS

DiffusionBee Reúne la descarga de modelos y las tareas de generación más habituales en una aplicación de escritorio. La documentación del proyecto enumera funciones como conversión imagen a imagen, relleno de áreas, ampliación de bordes, ControlNet, LoRA, SDXL y soporte para ciertos modelos Flux.

Elige esta opción cuando la instalación y una interfaz de usuario integrada sean más importantes que la exportación de un grafo pipeline. Antes de descargar un modelo de gran tamaño, verifica la versión más reciente y la arquitectura exacta, ya que la mera presencia de una etiqueta de funcionalidad como “soporte para Flux” no garantiza que se carguen todos sus derivados, componentes de cuantización ni elementos auxiliares.

ComfyUI: el flujo de trabajo es un artefacto

ComfyUI Representa la generación como un grafo de nodos. Un flujo de trabajo puede almacenarse como JSON de forma independiente de su resultado, lo que permite que el pipeline sea inspeccionable y versionable. Su ejecutor también puede evitar volver a calcular aquellas secciones del grafo cuyas entradas no han cambiado.

La versión actual de ComfyUI Desktop es compatible con Apple Silicon y gestiona su propio entorno, aunque la documentación oficial de macOS sigue etiquetándola como versión beta de Desktop. Existe una opción de instalación mediante Homebrew:

brew install comfyui

Elija ComfyUI cuando un pipeline cuente con ramas, componentes reutilizables, múltiples modelos o capacidades de automatización. De esta forma, podrá mantener unidos el JSON, la versión de la aplicación, las versiones de los nodos core, las revisiones de los nodos personalizados y el manifiesto del modelo.

Los nodos personalizados son dependencias ejecutables, y no presets inofensivos. Revise su código fuente, instale los scripts correspondientes, fije las revisiones relevantes e isole el entorno. Un grafo proveniente de un autor desconocido puede hacer referencia a códigos y modelos que aún no han sido auditados.

AUTOMATIC1111: compatibilidad con una interfaz de usuario web ya establecida

AUTOMATIC1111 Stable Diffusion WebUI Expone los parámetros de generación a través de una interfaz web y admite scripts, extensiones, así como un API. Elige esta opción cuando se requiere la integración de algún tutorial, automatización o extensión existente.

Su guía oficial para Apple Silicon describe las excepciones y limitaciones de rendimiento, incluyendo un bajo rendimiento en el entrenamiento. Asegúrese de que la funcionalidad requerida funcione en la versión de macOS y en las versiones de PyTorch objetivo antes de adoptar una configuración con numerosas extensiones.

Las extensiones ejecutan código dentro del entorno de la aplicación. Fíjelas, revise sus actualizaciones y mantenga el servicio en modo loopback, a menos que el acceso remoto sea intencionado y esté protegido.

Fooocus: una opción con límites propios de la era SDXL

Fooocus Oculta deliberadamente muchas decisiones técnicas detrás de un flujo de trabajo basado en una visión particular prompt. Esto puede resultar útil cuando sus valores predeterminados se ajustan a la tarea.

El README actual del proyecto indica que Fooocus está basado en SDXL y cuenta con un soporte a largo plazo limitado, únicamente para la corrección de errores. También señala que Mac no ha sido probado de forma exhaustiva y describe su instalación como no oficial. Por lo tanto, Fooocus constituye una opción restringida para flujos de trabajo existentes, y no representa la vía predeterminada para nuevas arquitecturas o infraestructuras macOS de larga duración.

Ejecutar una prueba de aceptación representativa

Utilice un conjunto de aceptación pequeño antes de decidirse por una herramienta:

  1. Reproducir un resultado base de texto a imagen a partir de una semilla y configuraciones registradas.
  2. Ejecutar la operación más compleja requerida: relleno de áreas, ControlNet, LoRA, ampliación de resolución o un grafo multietapa.
  3. Reiniciar la aplicación y reproducir el flujo de trabajo a partir de los artefactos guardados.
  4. Medir el tiempo de arranque en condiciones frías, el tiempo de generación, la presión máxima en la memoria y las dimensiones del resultado.
  5. Mover el flujo de trabajo a una cuenta de usuario o máquina limpia e identificar todas las dependencias faltantes.
  6. Verificar la vinculación a la red local, las descargas, la configuración de análisis, las licencias y el origen del modelo.

No se debe comparar herramientas que presenten diferentes checkpoints, resoluciones, cantidades de pasos o niveles de precisión, y denominar el resultado como un runtime benchmark.

Un valor por defecto práctico

Para el trabajo exploratorio, comience con una aplicación nativa y un modelo que la documente. Pase a ComfyUI cuando el propio flujo de trabajo adquiera valor: es decir, cuando sea necesario revisarlo, repetirlo, automatizarlo o entregárselo a otra persona. Mantenga AUTOMATIC1111 o Fooocus siempre que una dependencia existente haga útil su interfaz específica.

El activo duradero no es la captura de pantalla de un resultado satisfactorio. Se trata del paquete más reducido y reproducible que contiene las identidades del modelo, sus parámetros, dependencias y el estado del flujo de trabajo, y que permite generar nuevamente dicho resultado.
