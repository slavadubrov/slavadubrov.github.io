    Configuración del MacBook para ingeniería de IA: herramientas de macOS, Python, Docker y flujo de trabajo en la terminal

    Este es mi checklist para convertir una instalación reciente de macOS en una estación de trabajo para ingeniería de IA: herramientas de línea de comandos, Homebrew, Python, Docker, configuraciones predeterminadas de la terminal y ajustes del editor.

    TL;DR: Una configuración práctica de MacBook para ingeniería de IA se diseña intencionadamente de forma sencilla: Homebrew, uv, Docker o Colima, Git, configuración de la shell, herramientas de editor, utilidades locales para LLM/OCR, y una copia de seguridad reproducible de los archivos .dotfile. Optimice todo para acelerar la reconstrucción del entorno, no por novedad.

    1. Instalar las Herramientas de Línea de Comando de Xcode

    xcode-select --install

    Esto abre un diálogo que le guía paso a paso durante la instalación.

    Homebrew Es el gestor de paquetes que utilizo para todo lo demás de esta lista:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    El instalador solicita tu contraseña y se agrega a tu PATH## 3. Instalar las herramientas principales

    brew install openssl readline sqlite3 xz zlib pyenv uv htop gitmoji pandoc ncdu tmux

    Aproximadamente agrupados según su finalidad:

    Entorno Python

    • pyenv — gestionar múltiples versiones de Python en paralelo UV — gestor de paquetes y administrador de entornos rápido para Python

    Bibliotecas del sistema

    • openssl — Soporte para criptografía SSL/TLS lectura de línea — edición de texto en línea de comandos sqlite3 — base de datos incrustada ligera xz — compresión de datos zlib — biblioteca de compresión

    Herramientas de productividad

    • htop — Monitor del sistema visual y visor de procesos tmux — Gestionar múltiples sesiones de terminal ncdu — Analizar el uso del disco de forma interactiva gitmoji — Añadir emojis a los mensajes de commit pandoc — Convertir documentos entre formatos

    Para obtener más información al respecto uv en sí mismo, consulte el mío Guía rápida para gestionar Python en macOS con uv## 4. Elegir una terminal

    La Terminal de macOS por defecto es suficiente. Yo utilicé iTerm2 durante años y recientemente se mudó a Warpe, una terminal basada en Rust con funciones de IA integradas. Puedes obtener Warp desde su sitio web### Opcional: Configuración de iTerm2

    Si decides seguir utilizando iTerm2, estos son los dos parámetros que siempre modifico:

    Habilitar la edición de texto natural

    1. Preferencias → Perfiles → Teclas → Asignaciones de teclas
    2. Haz clic en el menú desplegable “Predefinidos…”
    3. Elige “Edición de texto natural”

    Elegir un tema de color

    1. Navega por los temas en iTerm2-Esquemas-de-Colores
    2. Preferencias → Perfiles → Colores → Predefiniciones de color…
    3. Haga clic en Importar y seleccione el tema que haya descargado

    5. Configurar Zsh con Oh My Zsh

    macOS utiliza Zsh de forma predeterminada. ¡Vaya con Zsh! Añade valores por defecto razonables y un sistema de plugins adicional:

    brew install zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

    El instalador realiza una copia de seguridad de tu configuración actual de Zsh antes de sustituirla por la nueva.

    6. Agregar complementos de Zsh

    Edita tu ~/.zshrc para cargar estos complementos:

    plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

    Las descripciones de todos estos se encuentran en el ¡Oh My Zsh! Wiki de complementosLos dos últimos requieren una instalación por separado:

    • zsh-autosuggestions — sugiere comandos a medida que escribes, en función de tu historial resaltado-de-sintaxis-zsh — Se colorean los comandos a medida que se escribe, por lo que los errores de escritura son evidentes.

    Sigue las instrucciones de instalación de cada repositorio.

    10. Esa es la configuración

    Eso es lo que instalo en cada MacBook nuevo antes de empezar a trabajar realmente. Ajustadlo según vuestros gustos.

    Puntos clave

    1. Haced que la configuración sea reproducible: documentad los comandos de instalación, los archivos .dotfile, las claves SSH, las extensiones del editor y las ubicaciones de los modelos/cache.
    2. Separad la gestión de paquetes del sistema de la gestión de proyectos en Python. Homebrew y uv resuelven problemas diferentes.
    3. Instalad las herramientas de IA locales únicamente después de que el entorno de desarrollo básico esté establecido.
    4. Una buena configuración de estación de trabajo se mide por la rapidez con la que podéis volver a crearla tras una instalación limpia de macOS.
