Configuración del MacBook para el desarrollo de AI: Herramientas de macOS, Python, Docker y flujo de trabajo en la Terminal

Esta es mi lista de verificación para convertir una instalación fresca de macOS en un puesto de trabajo de ingeniería AI: herramientas de línea de comandos, Homebrew, Python, Docker, configuraciones predeterminadas de la terminal y ajustes del editor.

TL;DR: Este es el entorno que utilizo en un nuevo MacBook: Homebrew, Python con uv, el Zsh incluido con Oh My Zsh y Powerlevel10k, Docker, mis editores y las herramientas locales AI. Es importante mantener los comandos y los archivos de configuración bajo control de versiones para que la configuración sea reutilizable.

1. Instalar las Herramientas de Línea de Comandos de Xcode

xcode-select --install

Esto abre un diálogo que le guía paso a paso durante la instalación.

2. Instalar Homebrew

Homebrew es el gestor de paquetes que utilizo para todo lo demás de esta lista:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

El instalador explica qué cambios se realizarán y solicita confirmación. En Apple Silicon, Homebrew utiliza /opt/homebrew de forma predeterminada. Siga los brew shellenv El comando se muestra al final; Homebrew no hace que dicho comando esté disponible de forma silenciosa en todos los shells.

3. Instalar las herramientas básicas

brew install openssl readline sqlite3 xz zlib uv htop gitmoji pandoc ncdu tmux

Aproximadamente agrupados según su finalidad:

Entorno de Python

Antes de adoptar uv, utilizaba pyenv como base para la gestión de mi entorno de Python. Actualmente, uv se encarga de ese flujo de trabajo diario por completo, incluyendo la instalación de Python y la creación de entornos de proyecto.

  • UV Instala versiones de Python y gestiona los entornos de proyecto, las dependencias, los comandos y los archivos de bloqueo. pyenv Es opcional. Solo lo utilizo cuando necesito un intérprete compilado a partir de fuentes o opciones de compilación personalizadas para CPython que las distribuciones ya preparadas por uv no ofrecen.

Instale pyenv por separado cuando necesite esa ruta de compilación desde el código fuente:

brew install pyenv

Bibliotecas del sistema

  • openssl Ofrece soporte para la criptografía SSL/TLS. entrada de línea Añade la posibilidad de editar texto desde la línea de comandos. sqlite3 Se trata de una base de datos embebida de bajo consumo.
  • xz y zlib Proporcionar compresión de datos.

Herramientas de productividad

  • htop Es un monitor de sistemas y procesos visual. tmux Gestiona múltiples sesiones de terminal. ncdu Analiza el uso del disco de forma interactiva.
  • gitmoji añade emojis a los mensajes de commit. pandoc Convierte documentos entre diferentes formatos.

Para obtener más información sobre uv en sí mismo, véase mi Guía rápida para gestionar Python en macOS con uv.

4. Elegir una terminal

El Terminal de macOS por defecto es suficiente. Yo utilicé iTerm2 durante años y recientemente se trasladó a Warpe, Una terminal basada en Rust que cuenta con funciones integradas de AI. La elección es personal; ninguna de las configuraciones posteriores depende de ella.

Opcional: configuración de iTerm2

Si sigues utilizando iTerm2, hay dos configuraciones que siempre modifico:

Habilitar la edición de texto natural

  1. Preferencias → Perfiles → Claves → Asignaciones de claves
  2. Haga clic en el menú desplegable “Predefinidos…”
  3. Seleccione “Edición de texto natural”

Elegir un tema de color

  1. Explorar temas en iTerm2-Esquemas-de-Colores
  2. Preferencias → Perfiles → Colores → Predefiniciones de color…
  3. Haga clic en Importar y seleccione el tema que haya descargado.

5. Configurar Zsh

macOS ya incluye Zsh y lo utiliza como shell de inicio de sesión predeterminado. Yo uso el que viene incluido. /bin/zsh; instalar Zsh de Homebrew Solo cuando sea necesario utilizar una versión específica más reciente del código fuente.

Compruebe el Zsh instalado y su shell de inicio de sesión actual:

echo "$SHELL"
command -v zsh
zsh --version

Si Zsh está instalado pero no está seleccionado como tu shell de inicio de sesión, cambia a la copia incluida:

chsh -s /bin/zsh

Abra un terminal nuevo tras cambiar la shell de inicio de sesión.

¡Vaya con Zsh! Añade encima los valores por defecto y el sistema de plugins que yo utilizo. Su instalador oficial es:

sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

El proyecto recomienda inspeccionar un instalador desconocido antes de ejecutarlo. También documenta que existe uno ya disponible. .zshrc se renombra a .zshrc.pre-oh-my-zsh durante la instalación.

6. Añadir complementos de Zsh

Edita tu ~/.zshrc para cargar estos complementos:

plugins=(
    aws bgnotify brew docker docker-compose
    emoji forklift gcloud git history iterm2
    keychain kubectl macos pre-commit
    pyenv pylint python screen themes
    tmux virtualenv vscode
    zsh-autosuggestions zsh-syntax-highlighting
)

Eliminar pyenv De la lista de complementos, si omitiste la instalación opcional de pyenv.

Las descripciones de todos estos se encuentran en el Oh My Zsh plugins wiki.

Los dos últimos requieren una instalación por separado:

  • zsh-autosuggestions Sugiere comandos de su historial mientras escribe. resaltado-de-sintaxis-zsh Se colorean los comandos a medida que se escriben, lo que facilita la detección de errores tipográficos.

Sigue las instrucciones de instalación de cada repositorio.

7. Aplicar el tema con Powerlevel10k

Powerlevel10k Es el tema de Zsh que uso. Muestra el directorio de trabajo, el estado de Git y el entorno de Python activo en el prompt, e incluye un asistente interactivo para ajustar su apariencia. Las instrucciones de instalación se encuentran en la página de GitHub.

Fuentes en otros editores

Si utilizas VSCode u otro editor con una terminal integrada, configura la fuente de la terminal para que los iconos de Powerlevel10k se muestren correctamente:

  1. Abra la configuración de su editor.
  2. Busque terminal.integrated.fontFamily
  3. Configúrelo en MesloLGS NF (instalado junto con Powerlevel10k)

La configuración completa de fuentes se encuentra en el Guía de fuentes Powerlevel10k.

9. El resto

Algunos complementos adicionales que instalo en cada máquina:

  • GitHub Desktop — cliente visual de Git cuando no quiero recurrir a la CLI Docker Desktop — el contenedor runtime que utilizo; Podman Desktop es una alternativa
  • Ollama o LM Studio — Ejecuta LLMs localmente en el Mac.

10. Esa es la configuración inicial

Eso es lo que instalo en cada MacBook nuevo antes de empezar a trabajar de verdad. Puedes ajustarlo según tus preferencias.

Conclusiones clave

  1. Se trata de una configuración personal, no de una línea de base mínima ni universal para macOS. Elimine todo aquello que no utilice.
  2. El paquete incluido /bin/zsh ¡Esto es suficiente para esta configuración! Instale la versión de Zsh proporcionada por Homebrew únicamente cuando necesite una versión específica más reciente proveniente del código fuente original.
  3. En mi flujo de trabajo diario, uv se encarga ahora de gestionar tanto la instalación de Python como los entornos de proyecto. pyenv sigue siendo opcional en el caso de interpretadores CPython compilados a partir del código fuente o personalizados.
  4. Guarde bien documentados los comandos de instalación, los archivos de configuración sin datos sensibles, las extensiones del editor y las ubicaciones de los modelos, de modo que la siguiente configuración sea un proceso mecánico y sencillo.
